## Slide 2

**핵심 개념**
이 슬라이드는 데이터분석 입문 (CSED226) 강의의 한 세션인 **의사결정 나무 (Decision Trees)** 모델에 대한 학습 로드맵을 제시합니다. 의사결정 나무는 지도 학습 알고리즘 중 하나로, 분류(Classification)와 회귀(Regression) 문제 해결에 사용됩니다. 이 로드맵은 의사결정 나무의 기본 개념부터 복잡성, 모델 구축 방법, 성능 평가 지표(Entropy, Information Gain, Gini Impurity) 및 실제 적용 예시에 이르기까지 포괄적인 학습 내용을 다룰 것임을 보여줍니다. 특히, 나무를 구성하는 핵심 원리인 **탐욕적 분할(Greedy Splitting)**과 관련된 개념들을 자세히 다룰 예정입니다.

**코드/수식 해설**
이 슬라이드 자체에는 코딩이나 수식이 없지만, 로드맵에 언급된 개념들을 바탕으로 다음과 같은 핵심 수식과 코드 예시가 다루어질 것으로 예상됩니다.

*   **Entropy (엔트로피)**: 데이터 집합 $S$의 불확실성 또는 무질서도를 측정하는 척도입니다. 클래스 $i$의 비율을 $p_i$라고 할 때:
    $$H(S) = -\sum_{i=1}^c p_i \log_2(p_i)$$
    클래스 수 $c$가 2개인 이진 분류의 경우, $p_+$를 양성 클래스의 비율, $p_-$를 음성 클래스의 비율이라 하면:
    $$H(S) = -p_+ \log_2(p_+) - p_- \log_2(p_-)$$

*   **Information Gain (정보 이득)**: 특정 특성(attribute) $A$를 사용하여 데이터를 분할했을 때 얻을 수 있는 불확실성 감소량입니다. 엔트로피를 기반으로 계산되며, 분할 전 엔트로피에서 분할 후 엔트로피의 가중 평균을 뺀 값입니다.
    $$IG(S, A) = H(S) - \sum_{v \in Values(A)} \frac{|S_v|}{|S|} H(S_v)$$
    여기서 $Values(A)$는 특성 $A$가 가질 수 있는 모든 값의 집합, $S_v$는 특성 $A$의 값이 $v$인 $S$의 부분집합입니다.

*   **Gini Impurity (지니 불순도)**: 엔트로피와 유사하게 데이터 집합의 불순도(impurity)를 측정하는 척도입니다. 계산 비용이 엔트로피보다 낮아 CART(Classification and Regression Trees) 알고리즘에서 주로 사용됩니다.
    $$G(S) = 1 - \sum_{i=1}^c p_i^2$$

# 범주형 데이터를 수치형으로 인코딩 (Decision Tree는 수치형 데이터를 선호)

dt_classifier.fit(X_train, y_train)

# 예측 및 평가

accuracy = accuracy_score(y_test, y_pred)
print(f"Decision Tree Accuracy: {accuracy:.2f}")

## Slide 3

**핵심 개념**
*   **분류(Classification) 문제**: 주어진 데이터의 특징(features)을 기반으로 해당 데이터가 어떤 범주(클래스)에 속하는지 예측하는 머신러닝 문제의 한 유형입니다. 슬라이드에서는 레몬과 오렌지를 구분하는 예시를 통해 이를 설명합니다.
*   **특징(Features)**: 데이터를 분류하는 데 사용되는 속성들입니다. 여기서는 과일의 `width (cm)`와 `height (cm)`를 특징으로 사용합니다. `rougher skin`과 같은 다른 특징도 고려될 수 있지만, 초기 단계에서는 수치형 특징에 집중합니다.
*   **의사결정 트리(Decision Tree)의 기본 아이디어**: 특징 공간(feature space)을 축에 정렬된(axis-aligned) 직사각형 영역으로 분할하여 데이터를 분류하는 모델입니다. 각 분할은 특정 특징에 대한 임계값(threshold) 비교를 통해 이루어집니다.
*   **판별 기준(Decision Rule)**: 각 클래스를 구분하는 규칙입니다. 레몬은 `height > width`로 길쭉한 형태, 오렌지는 `height ≈ width`로 둥근 형태라는 직관적인 특징을 기반으로 합니다.

**코드/수식 해설**
슬라이드에서 제시된 판별 기준 예시는 의사결정 트리의 분할 조건을 나타냅니다.
*   $height \le \tau$: 높이가 특정 임계값($\tau$)보다 작거나 같은지 확인하는 조건입니다.
*   $width - height \le \tau$: 너비와 높이의 차이가 특정 임계값($\tau$)보다 작거나 같은지 확인하는 조건입니다.
이러한 조건들은 의사결정 트리의 노드에서 데이터를 두 개의 자식 노드로 분할하는 데 사용됩니다.

**구체적 예시**
레몬과 오렌지 분류를 위한 `Toy sample` 데이터는 다음과 같습니다:

```
Fruit   width   height
lemon   5.2     7.6
lemon   5.0     7.2
orange  6.8     6.9
orange  7.1     7.0
```

이 예시 데이터를 보면:
*   첫 번째 레몬(`width=5.2`, `height=7.6`)의 경우, `height`가 `width`보다 훨씬 큽니다 (7.6 > 5.2). `height - width = 2.4`.
*   두 번째 레몬(`width=5.0`, `height=7.2`)의 경우도 `height`가 `width`보다 큽니다 (7.2 > 5.0). `height - width = 2.2`.
*   첫 번째 오렌지(`width=6.8`, `height=6.9`)의 경우, `height`와 `width`가 거의 비슷합니다 (6.9 ≈ 6.8). `height - width = 0.1`.
*   두 번째 오렌지(`width=7.1`, `height=7.0`)의 경우도 `height`와 `width`가 거의 비슷합니다 (7.0 ≈ 7.1). `height - width = -0.1`.

이 데이터를 통해 `height - width` 값이 양수이고 큰 경우 레몬일 가능성이 높고, 0에 가까운 경우 오렌지일 가능성이 높다는 간단한 규칙을 도출할 수 있습니다. 이는 의사결정 트리가 어떻게 특징값을 활용하여 분류 규칙을 학습하는지 보여주는 기초적인 예시입니다.

**시험 포인트**
*   ⭐ **분류 문제의 본질**: 주어진 특징을 바탕으로 데이터를 특정 클래스로 할당하는 것이 분류 문제의 목표임을 정확히 이해해야 합니다.
*   ⭐ **의사결정 트리의 동작 원리**: 의사결정 트리가 특징 공간을 축에 정렬된 직사각형 영역으로 분할하여 분류를 수행한다는 핵심 아이디어를 반드시 숙지해야 합니다. (Axis-aligned split)
*   ⭐ **특징(Feature)의 중요성**: 분류 모델의 성능은 어떤 특징을 사용하고, 그 특징들을 어떻게 활용하여 판별 기준을 설정하는지에 따라 크게 달라질 수 있음을 이해해야 합니다.

**핵심 개념**

*   **결정 트리 (Decision Tree) 분류**: 결정 트리는 이진 트리 구조를 가지며, 각 내부 노드는 특정 특성(feature)에 대한 조건(predicate, 주로 임계값)을 적용하여 데이터를 분할합니다. 트리의 최하단에 있는 리프(leaf) 노드들은 최종적으로 클래스 레이블 또는 클래스 분포를 예측합니다.
*   **가설 클래스 (Hypothesis class)**: 결정 트리는 입력 공간을 축에 정렬된(axis-aligned) 직사각형 영역으로 분할하고, 각 영역 내에서는 일정한(constant) 예측값을 가지는 '조각별 상수 분류기(piecewise-constant classifier)'의 한 종류입니다.
*   **리프 예측 (Leaf prediction)**: 리프 노드에 도달한 데이터 포인트들은 해당 리프 노드에 속한 데이터들 중 가장 많은 비중을 차지하는 클래스(majority class)로 예측되거나, 각 클래스에 속할 사후 확률(posterior probability) $p̂(y \mid \text{leaf})$을 기반으로 예측됩니다.
*   **리프 불순도 (Leaf impurity)**: 결정 트리는 데이터를 분할할 때, 각 분할 이후 자식 노드들의 '불순도'를 최소화하는 방향으로 최적의 분할 지점을 찾습니다. 불순도는 노드 내에 여러 클래스가 얼마나 섞여 있는지를 나타내는 척도이며, 불순도가 낮을수록 노드는 '순수'하다고 할 수 있습니다.

**코드/수식 해설**

결정 트리 학습 과정에서 분할의 품질을 평가하기 위해 다음 두 가지 불순도 척도가 주로 사용됩니다. 여기서 $p_k$는 주어진 노드에서 클래스 $k$에 속하는 데이터의 비율을 의미합니다.

*   **엔트로피 (Entropy)**: 불확실성 또는 무질서도를 측정하는 척도입니다. 엔트로피 값이 높을수록 노드 내 클래스의 분포가 균등하여 불순도가 높다는 것을 의미합니다.
    $$ \text{Entropy}(\mathbf{p}) = - \sum_{k} p_k \log_2 p_k $$
    -   예시: 두 개의 클래스(0과 1)가 반반 섞여 있다면($p_0=0.5, p_1=0.5$), Entropy는 $-(0.5 \log_2 0.5 + 0.5 \log_2 0.5) = -(0.5 \times -1 + 0.5 \times -1) = 1$로 최대값을 가집니다. 한 클래스만 있다면($p_0=1, p_1=0$), Entropy는 $-(1 \log_2 1 + 0 \log_2 0) = 0$ (여기서 $0 \log_2 0 = 0$으로 정의)으로 최소값을 가집니다.

*   **지니 불순도 (Gini impurity)**: 랜덤하게 선택한 데이터 포인트가 잘못 분류될 확률을 나타냅니다. 지니 불순도 또한 값이 높을수록 불순도가 높다는 것을 의미합니다.
    $$ \text{Gini}(\mathbf{p}) = \sum_{k} p_k (1 - p_k) $$
    -   예시: 두 개의 클래스가 반반 섞여 있다면($p_0=0.5, p_1=0.5$), Gini는 $0.5(1-0.5) + 0.5(1-0.5) = 0.25 + 0.25 = 0.5$로 최대값을 가집니다. 한 클래스만 있다면($p_0=1, p_1=0$), Gini는 $1(1-1) + 0(1-0) = 0$으로 최소값을 가집니다.

분할(splits)은 이 불순도(impurity)를 줄이는 방향으로 선택됩니다. 즉, 분할 후 자식 노드들의 불순도가 부모 노드보다 낮아지도록 하는 분할 지점을 찾습니다.

**구체적 예시**

어떤 사람이 대출을 받을 수 있을지(승인/거절) 예측하는 결정 트리를 가정해봅시다.

1.  **내부 노드의 분할**: 트리의 첫 번째 노드는 "신용 등급이 5등급 이상인가?"라는 질문으로 시작할 수 있습니다.
    *   "예"인 사람들은 왼쪽 자식 노드로, "아니오"인 사람들은 오른쪽 자식 노드로 이동합니다.
2.  **리프 노드의 예측**:
    *   왼쪽 노드로 간 사람들(신용 등급 5등급 이상)은 다시 "월 소득이 300만원 이상인가?"로 분할될 수 있습니다.
    *   만약 어떤 리프 노드에 도달한 100명 중 90명이 대출 승인을 받았고 10명만 거절되었다면, 이 리프 노드는 '대출 승인'으로 예측하게 됩니다 (majority class). 또는 대출 승인 확률 90%, 거절 확률 10%로 예측합니다.
3.  **불순도 감소**: 처음 모든 사람이 섞여 있는 상태(대출 승인과 거절이 반반)는 불순도가 높습니다. "신용 등급"으로 분할한 후, '신용 등급이 좋은 그룹'에는 승인된 사람이 많고, '신용 등급이 낮은 그룹'에는 거절된 사람이 많아진다면, 각 그룹의 불순도가 초기보다 낮아지게 됩니다. 결정 트리는 이렇게 불순도를 가장 크게 감소시키는 질문(특성과 임계값)을 찾아 분할을 수행합니다.

**시험 포인트**

*   ⭐ **결정 트리의 기본 구조**: 내부 노드(분할 조건), 리프 노드(예측)의 역할을 이해해야 합니다. 특히, 내부 노드의 조건이 주로 '단일 특성에 대한 임계값'이라는 점과 이것이 '축 정렬 분할'로 이어진다는 점을 기억하세요.
*   ⭐ **가설 클래스의 특징**: 결정 트리가 데이터 공간을 직사각형 영역으로 나누고 각 영역에서 상수 값을 예측하는 '조각별 상수 분류기'라는 개념을 파악해야 합니다.
*   ⭐ **불순도(Impurity)의 개념 및 중요성**: 불순도가 무엇이며, 왜 결정 트리 학습에서 불순도 감소가 핵심 원리인지 이해해야 합니다.
*   ⭐ **엔트로피 및 지니 불순도 수식**: 각 수식의 형태와 의미를 정확히 알고 있어야 합니다. 주어진 클래스 비율 $\mathbf{p}$에 대해 두 불순도 값을 계산할 수 있어야 합니다.
    *   $\text{Entropy}(\mathbf{p}) = - \sum_{k} p_k \log_2 p_k$
    *   $\text{Gini}(\mathbf{p}) = \sum_{k} p_k (1 - p_k)$
*   ⭐ **$0 \log_2 0 = 0$ 정의**: 엔트로피 계산 시, 특정 클래스의 비율이 0일 때 해당 항은 0으로 처리된다는 것을 기억해야 합니다.

**핵심 개념**:
*   **회귀(Regression)를 위한 의사결정 나무(Decision Tree)**: 이 슬라이드는 연속적인 수치 예측(예: 집값, 주가)을 목표로 하는 의사결정 나무의 작동 방식을 설명합니다. 분류(Classification) 트리와 달리, 회귀 트리는 각 리프 노드에 실수 값을 할당합니다.
*   **리프 노드(Leaf Node) 예측**: 특정 리프 노드에 도달한 모든 훈련 데이터 포인트들의 타겟 값($y_i$) 평균을 해당 리프 노드의 예측값으로 사용합니다. 이를 '후방 평균(posterior mean)'이라고 합니다.
*   **노드 불순도(Impurity)**: 회귀 트리에서 노드의 불순도는 해당 노드에 속한 데이터 포인트들의 타겟 값 분산(variance)을 나타냅니다. 주로 평균 제곱 오차(Mean Squared Error, MSE) 또는 제곱 오차 합(Sum of Squared Errors, SSE)으로 측정하며, 불순도가 낮을수록 노드 내 데이터의 타겟 값들이 서로 비슷하여 '순수'하다고 판단합니다.
*   **분할 기준(Split Criterion)**: 의사결정 나무가 노드를 분할할 때, 자식 노드(L, R)들의 불순도(SSE 또는 가중 MSE) 합을 가장 많이 감소시키는 분할 기준(특성과 임계값)을 선택합니다. 목표는 분할을 통해 데이터의 동질성을 최대로 높이는 것입니다.

**코드/수식 해설**:
*   **데이터셋 정의**: 훈련 데이터셋 $\mathcal{D}$는 $n$개의 샘플로 구성되며, 각 샘플은 특성 벡터 $x_i$와 실수 값 타겟 $y_i$의 쌍입니다.
    $$\mathcal{D} = \{(x_i, y_i)\}_{i=1}^n \text{ with } y_i \in \mathbb{R}$$
    노드 $\ell$에 도달하는 데이터 포인트들의 인덱스 집합을 $I_{\ell}$로, 그 개수를 $n_{\ell}$로 정의합니다.
*   **리프 노드 예측($\hat{y}_{\ell}$)**:
    리프 노드 $\ell$의 예측값 $\hat{y}_{\ell}$은 해당 노드에 속한 모든 데이터 포인트 $y_i$ 값의 평균입니다.
    $$\hat{y}_{\ell} = \frac{1}{n_{\ell}} \sum_{i \in I_{\ell}} y_i$$
*   **노드 $\ell$의 불순도 (MSE)**:
    노드 $\ell$의 불순도는 해당 노드에 속한 데이터 포인트들의 실제 $y_i$ 값과 노드 예측값 $\hat{y}_{\ell}$ 사이의 평균 제곱 오차(MSE)로 계산됩니다.
    $$\text{MSE}(\ell) = \frac{1}{n_{\ell}} \sum_{i \in I_{\ell}} (y_i - \hat{y}_{\ell})^2$$
    동일하게, 제곱 오차 합(SSE)으로도 표현할 수 있습니다.
    $$\text{SSE}(\ell) = \sum_{i \in I_{\ell}} (y_i - \hat{y}_{\ell})^2$$
    여기서 $\text{SSE}(\ell) = n_{\ell} \cdot \text{MSE}(\ell)$ 관계가 성립합니다. SSE는 MSE에 노드의 샘플 수를 곱한 값입니다.
*   **분할 기준**:
    후보 분할(candidate split)이 두 개의 자식 노드 $L$과 $R$을 생성할 때, 최적의 분할은 다음 값을 최소화하는 분할입니다.
    $$\text{SSE}(L) + \text{SSE}(R)$$
    이는 다음과 동등한 표현입니다.
    $$\frac{n_L}{n_L + n_R}\text{MSE}(L) + \frac{n_R}{n_L + n_R}\text{MSE}(R)$$
    여기서 $n_L$과 $n_R$은 각각 자식 노드 $L$과 $R$에 속하는 데이터 포인트의 개수입니다. 이 기준은 분할 후 자식 노드들의 총 불순도(제곱 오차 합)를 최소화하여 예측 성능을 향상시키는 것을 목표로 합니다.

**구체적 예시**:
아파트 가격을 예측하는 회귀 트리를 만든다고 가정해 봅시다.
*   **리프 노드 예측**: 어떤 리프 노드에 '강남, 30평대, 신축' 아파트 100개의 데이터가 도달했습니다. 이 100개 아파트의 실제 거래 가격 평균이 20억이라면, 이 리프 노드는 새로 입력된 '강남, 30평대, 신축' 아파트에 대해 20억을 예측값으로 내놓습니다.
*   **노드 불순도**: 위 100개 아파트의 실제 가격이 모두 20억에 아주 가깝다면(예: 19.5억~20.5억), 해당 노드의 MSE는 매우 낮습니다. 하지만 가격이 10억부터 30억까지 넓게 퍼져 있다면 MSE는 높아지며, 이 노드는 '불순하다'고 판단되어 추가적인 분할이 필요할 수 있습니다.
*   **분할 기준**: 현재 노드를 '지하철역과의 거리'라는 특성으로 분할하려 합니다. '5분 이내' 아파트는 노드 L로, '5분 초과' 아파트는 노드 R로 보냅니다. 분할 후 노드 L과 R의 SSE를 각각 계산하여 합산합니다. 만약 '층수'라는 다른 특성으로 분할했을 때의 SSE 합이 더 작다면, '층수'를 기준으로 분할을 선택합니다. 즉, 분할 후 각 그룹(자식 노드) 내에서 가격 편차가 가장 작아지는 분할을 찾는 것입니다.

**시험 포인트**:
*   ⭐ **회귀 트리의 리프 노드 예측값**: 리프 노드에 속한 샘플들의 타겟 값 **평균**이라는 것을 정확히 이해해야 합니다.
*   ⭐ **회귀 트리의 노드 불순도 측정 지표**: MSE 또는 SSE를 사용하며, 이들이 노드 내 타겟 값들의 **분산**을 나타낸다는 의미를 알아야 합니다.
*   ⭐ **회귀 트리의 최적 분할 기준**: 자식 노드들의 **SSE 합을 최소화**하는 분할을 선택한다는 것을 기억해야 합니다. 이는 가중 MSE 합을 최소화하는 것과 동등합니다.
*   MSE와 SSE의 관계($\text{SSE}(\ell) = n_{\ell} \cdot \text{MSE}(\ell)$)를 이해하고 계산에 적용할 수 있어야 합니다.
*   회귀 의사결정 나무가 분류 의사결정 나무와 예측 방식 및 불순도 측정 방식에서 어떻게 다른지 비교하여 설명할 수 있어야 합니다.

### **Finding the Smallest Tree is NP-hard**

---

*   **핵심 개념**:
    *   **최소 크기 의사결정 트리 문제 (Minimum-size Decision Tree Problem)**: 주어진 레이블된 데이터에 대해 모든 데이터 포인트를 정확하게 분류(consistent)하면서 가장 적은 수의 노드(분기 또는 리프)를 가지는 의사결정 트리를 찾는 문제입니다.
    *   **NP-hard**: 이 문제는 계산 복잡도 이론에서 NP-hard 클래스에 속합니다. 이는 알려진 다항 시간(polynomial time) 알고리즘이 없으며, 문제의 크기(데이터의 특징 수나 샘플 수)가 증가함에 따라 최적해를 찾는 데 필요한 시간이 기하급수적으로 증가하여 현실적으로 해결 불가능해진다는 의미입니다. 즉, 전역 최적화(global optimization)를 통해 최적의 트리를 찾는 것은 거의 불가능합니다.
    *   **실용적 접근**: NP-hard 문제의 특성상, 실제 의사결정 트리 알고리즘(예: CART, C4.5)은 전역 최적화 대신 '탐욕적 휴리스틱(greedy heuristics)'과 '가지치기(pruning)' 기법을 사용합니다. 이는 최적해를 보장하지는 않지만, 합리적인 시간 내에 좋은 성능을 내는 트리를 구축할 수 있게 합니다.

*   **코드/수식 해설**:
    이 슬라이드에는 직접적인 코드나 수식은 포함되어 있지 않습니다. 하지만 NP-hard의 개념은 알고리즘의 시간 복잡도($O(f(n))$)와 관련이 깊습니다. 다항 시간($O(n^k)$)이 아닌 지수 시간($O(k^n)$) 이상의 복잡도를 가질 수 있음을 내포합니다.
    최소 크기 의사결정 트리를 찾는 문제 $P$가 NP-hard라는 것은, 임의의 NP 문제 $Q$가 $P$로 다항 시간 내에 환원(reduction)될 수 있다는 것을 의미합니다.

*   **구체적 예시**:
    *   **NP-hard 비유**: 여러분이 여행 가방(의사결정 트리)에 필요한 모든 물건(데이터 포인트)을 완벽하게 담으면서, 가방의 부피(트리의 크기)를 최소화하고 싶다고 가정해 봅시다. 물건들을 넣는 모든 가능한 순서와 조합을 고려하는 것은 너무 많은 시간이 걸려서 불가능합니다. 마치 바둑판에 모든 가능한 수를 계산하는 것과 유사합니다.
    *   **탐욕적 휴리스틱 비유**: 대신, 여러분은 "가장 부피가 큰 옷부터 넣자" 또는 "가장 중요한 물건부터 넣자"와 같은 간단한 규칙(탐욕적 휴리스틱)을 사용하여 가방을 쌉니다. 각 단계에서 현재 상황에서 가장 좋아 보이는 선택을 하는 것이죠. 이렇게 하면 가방을 빨리 쌀 수 있지만, 이것이 항상 가방 부피를 최소화하는 최적의 방법은 아닐 수 있습니다.
    *   **가지치기 비유**: 가방을 다 싼 후에, 불필요하게 공간을 차지하는 작은 액세서리들(트리의 불필요한 가지나 노드)을 빼거나 재배치하여 공간을 더 효율적으로 만드는 과정이 가지치기에 해당합니다. 이는 복잡도를 줄이고 일반화 성능을 향상시키는 데 도움이 됩니다.

*   **시험 포인트**:
    *   ⭐ 최소 크기 의사결정 트리를 찾는 문제가 NP-hard라는 사실과 그 의미를 정확히 이해해야 합니다.
    *   ⭐ 실제 의사결정 트리 알고리즘이 왜 전역 최적화 대신 탐욕적 휴리스틱과 가지치기를 사용하는지 그 이유를 설명할 수 있어야 합니다.
    *   ⭐ NP-hard 문제의 특징과 실제 머신러닝 문제 해결에서의 제약사항 및 대안적 접근 방식(근사 알고리즘)에 대해 질문할 수 있습니다.

**핵심 개념**:
이 슬라이드는 의사결정 트리를 구축하는 기본적인 탐욕적(Greedy) 휴리스틱 접근 방식인 **상향식 의사결정 트리 유도 (Top-down induction of decision trees, TDIDT)**와 트리를 분할할 때 사용되는 **불순도 감소 (Impurity Reduction)** 목표 함수를 설명합니다.

1.  **상향식 의사결정 트리 유도 (TDIDT)**:
    *   트리를 재귀적으로 위에서 아래로(루트에서 시작하여 리프 노드로) 성장시키는 알고리즘입니다.
    *   각 단계에서 현재 노드를 분할할지, 아니면 리프 노드로 만들지 결정합니다.
    *   **리프 노드 조건**: 노드의 모든 샘플이 동일한 클래스에 속하여 "순수(pure)"하거나, 미리 정의된 "정지 조건(stopping criterion)"을 만족하면 해당 노드를 더 이상 분할하지 않고 리프 노드로 만듭니다.
    *   **분할 과정**: 리프 노드 조건에 해당하지 않는 경우, 현재 노드에서 가능한 모든 잠재적 분할(candidate split)을 고려하고, 각 분할이 가져올 **불순도 감소량**을 계산합니다.
    *   **최적 분할 선택**: 계산된 불순도 감소량이 가장 큰 분할을 선택하고, 해당 분할로 생성된 자식 노드에 대해 이 과정을 재귀적으로 반복합니다. 이것이 "탐욕적(Greedy)"이라는 표현이 사용되는 이유입니다. 즉, 각 단계에서 현재 상황에서 가장 좋은 선택을 합니다.

2.  **정지 조건 (Stopping criteria)**:
    *   트리의 과적합(overfitting)을 방지하고 복잡도를 제어하기 위한 규칙입니다. 일반적인 정지 조건으로는 트리의 최대 깊이(max depth), 리프 노드에 포함될 최소 샘플 수(min samples per leaf), 최소 불순도 감소량(min impurity decrease) 등이 있습니다.

3.  **불순도 감소 (Impurity Reduction)**:
    *   의사결정 트리가 분할을 결정하는 핵심 기준입니다. 노드가 분할되기 전의 불순도에서 분할된 후 자식 노드들의 가중 평균 불순도를 뺀 값입니다. 이 값이 클수록 더 좋은 분할로 간주됩니다. 불순도(Impurity)는 일반적으로 지니 불순도(Gini impurity)나 엔트로피(Entropy)를 사용하여 측정됩니다.

**코드/수식 해설**:
슬라이드에 제시된 "Local objective"는 주어진 노드에서 최적의 분할 $s^*$를 찾는 수학적 표현입니다.

$$
s^* = \arg \max_s \Delta I(s) \quad \text{where} \quad \Delta I(s) = I(\hat{p}_{parent}) - \sum_{z \in \{L,R\}} w_z I(\hat{p}_{Z=z})
$$

*   $s^*$: 현재 노드에서 불순도 감소 $\Delta I(s)$를 최대화하는 최적의 분할(split)을 나타냅니다.
*   $\arg \max_s$: 주어진 함수 $\Delta I(s)$를 최대화하는 인자 $s$를 찾는 연산자입니다.
*   $\Delta I(s)$: 특정 분할 $s$가 발생했을 때 얻을 수 있는 불순도 감소량입니다.
*   $I(\hat{p}_{parent})$: 분할이 이루어지기 전 부모 노드의 불순도입니다. $\hat{p}_{parent}$는 부모 노드의 클래스 분포를 의미합니다.
*   $\sum_{z \in \{L,R\}} w_z I(\hat{p}_{Z=z})$: 분할 후 생성된 자식 노드(왼쪽 $L$, 오른쪽 $R$)들의 불순도를 가중 평균한 값입니다.
    *   $z \in \{L,R\}$: 왼쪽(L) 또는 오른쪽(R) 자식 노드를 나타냅니다.
    *   $w_z$: 각 자식 노드 $z$의 가중치로, 일반적으로 부모 노드에 있던 전체 샘플 중 해당 자식 노드로 이동한 샘플의 비율입니다. 예를 들어, $w_L = \frac{\text{샘플 수}(L)}{\text{샘플 수}(parent)}$입니다.
    *   $I(\hat{p}_{Z=z})$: 각 자식 노드 $z$의 불순도입니다. $\hat{p}_{Z=z}$는 자식 노드 $z$의 클래스 분포를 의미합니다.

**구체적 예시**:
환자의 질병 진단을 위한 의사결정 트리를 구축한다고 가정해 봅시다. 한 노드에 "기침" 증상을 가진 환자 100명(50명은 감기, 50명은 독감)이 있다고 합시다.
*   이 노드의 불순도는 매우 높습니다 (50:50).
*   여기서 "열이 나는가?"를 기준으로 분할을 시도합니다.
    *   '예' 그룹: 환자 60명 (감기 10명, 독감 50명)
    *   '아니오' 그룹: 환자 40명 (감기 40명, 독감 0명)
*   분할 전 부모 노드의 불순도 $I(\hat{p}_{parent})$를 계산합니다. (예: Gini impurity $1 - (0.5^2 + 0.5^2) = 0.5$)
*   분할 후 '예' 그룹 불순도 $I(\hat{p}_{Yes})$와 '아니오' 그룹 불순도 $I(\hat{p}_{No})$를 각각 계산합니다. (예: $I(\hat{p}_{Yes}) = 1 - (1/6^2 + 5/6^2) \approx 0.278$, $I(\hat{p}_{No}) = 1 - (1^2 + 0^2) = 0$)
*   각 그룹의 가중치 $w_{Yes} = 60/100 = 0.6$, $w_{No} = 40/100 = 0.4$를 적용하여 가중 평균 불순도를 계산합니다.
    $(0.6 \times 0.278) + (0.4 \times 0) \approx 0.167$
*   불순도 감소 $\Delta I(s) = 0.5 - 0.167 = 0.333$ 입니다.

다른 분할 ("두통이 있는가?")을 시도했을 때 불순도 감소량이 0.2로 나왔다면, 트리는 "열이 나는가?" 분할을 선택할 것입니다. 이는 가장 큰 불순도 감소를 가져오기 때문입니다.

**시험 포인트**:
*   ⭐**TDIDT의 기본 원리**: 의사결정 트리가 어떻게 재귀적으로 성장하는지, 특히 "탐욕적(Greedy)" 접근 방식의 의미를 이해해야 합니다.
*   ⭐**불순도 감소 ($\Delta I(s)$)의 역할과 계산 방식**: 의사결정 트리가 최적의 분할을 선택하는 기준이 불순도 감소라는 것을 이해하고, 주어진 수식의 각 항이 무엇을 의미하는지 설명할 수 있어야 합니다. (예: Gini impurity 또는 Entropy와 같은 불순도 함수가 주어졌을 때 $\Delta I(s)$ 계산)
*   ⭐**정지 조건의 중요성**: 트리의 과적합 방지와 복잡도 제어를 위해 정지 조건(최대 깊이, 최소 샘플 수 등)이 왜 필요한지 설명할 수 있어야 합니다.
*   **"순수 노드"의 의미**: 특정 노드의 모든 샘플이 동일한 클래스에 속하는 상태를 이해해야 합니다.

---## Slide 8

**핵심 개념**

*   **지역적 탐욕적 목표 (Local Greedy Objective)**: 의사결정 트리가 노드를 분할할 때 사용하는 기본 전략입니다. 각 내부 노드에서 다음 최적의 특성(feature)을 선택하여 분할을 수행합니다. "최적"의 기준은 해당 분할이 데이터의 불순도(impurity)를 가장 많이 감소시키거나, 동등하게 정보 이득(information gain)을 가장 많이 최대화하는 것입니다. 즉, 전체 트리의 최적화가 아닌, 현재 노드에서 가장 좋은 선택을 반복적으로 하는 탐욕적인 방식입니다.
*   **정보 이득 (Information Gain)**: 분할 전 부모 노드의 불확실성(엔트로피)에서 분할 후 자식 노드들의 가중 평균 불확실성(엔트로피)을 뺀 값입니다. 이 값이 클수록 해당 특성으로 데이터를 분할했을 때 클래스 예측의 순수도(purity)가 더 높아짐을 의미합니다.
*   **불순도 감소 (Impurity Reduction)**: 정보 이득과 같은 개념으로, 특정 특성으로 분할을 했을 때 노드의 불순도(예: 엔트로피 또는 지니 불순도)가 얼마나 감소하는지를 측정합니다. 불순도 감소량이 최대가 되는 특성과 분할 지점을 선택하여 트리를 성장시킵니다.
*   **이진 분할 (Binary Split)**: 대부분의 의사결정 트리 알고리즘(예: CART)은 각 노드를 두 개의 자식 노드로 분할합니다. 슬라이드의 수식은 이러한 이진 분할을 가정합니다 ($Z \in \{L, R\}$).

**코드/수식 해설**

의사결정 트리에서 최적의 분할 $s^*$를 선택하는 수식은 다음과 같습니다.
여기서 $s$는 후보 분할(candidate split)을 의미합니다.

$$
s^* = \arg \max_s IG(Y; Z_s) = \arg \max_s \left[H(Y; \hat{p}_{parent}) - \sum_{z \in \{L, R\}} w_z H(Y; \hat{p}_{Z=z})\right]
$$
*   $\mathbf{s^*}$: 최적의 분할 기준 (예: '특성 X의 값이 5보다 작다'와 같은 조건)을 나타냅니다.
*   $\mathbf{\arg \max_s}$: 모든 가능한 분할 $s$ 중에서 괄호 안의 값을 최대로 만드는 $s$를 선택하라는 의미입니다.
*   $\mathbf{IG(Y; Z_s)}$: 분할 $s$에 의해 유도된 자식 노드들 $Z_s$와 타겟 변수 $Y$ 간의 정보 이득입니다.
*   $\mathbf{H(Y; \hat{p}_{parent})}$: 분할 전 부모 노드에 있는 데이터의 클래스 분포 $\hat{p}_{parent}$에 대한 엔트로피입니다. 이는 부모 노드의 불확실성을 나타냅니다.
*   $\mathbf{\sum_{z \in \{L, R\}} w_z H(Y; \hat{p}_{Z=z})}$: 분할 $s$에 의해 생성된 왼쪽(L)과 오른쪽(R) 자식 노드들의 엔트로피를 각각의 가중치 $w_z$로 평균한 값입니다.
    *   $w_z = n_z / n$: 자식 노드 $z$로 가는 데이터 수($n_z$)를 부모 노드의 전체 데이터 수($n$)로 나눈 비율입니다.
    *   $\hat{p}_{Z=z}$: 자식 노드 $z$에 있는 데이터의 클래스 분포입니다.
*   **해석**: 이 수식은 부모 노드의 불확실성(엔트로피)에서 자식 노드들의 평균 불확실성을 뺀 값이 가장 커지는 분할을 찾는 것입니다. 즉, 분할을 통해 불확실성이 가장 많이 감소하는 지점을 찾는 과정입니다.

**(ID3/C4.5 형태)**: 정보 이득의 일반적인 형태는 다음과 같습니다.
$$
\arg \max_i IG(X_i) = \arg \max_i (H(Y) - H(Y|X_i))
$$
*   $\mathbf{IG(X_i)}$: 특성 $X_i$에 대한 정보 이득입니다.
*   $\mathbf{H(Y)}$: 타겟 변수 $Y$의 엔트로피입니다.
*   $\mathbf{H(Y|X_i)}$: 특성 $X_i$가 주어졌을 때 타겟 변수 $Y$의 조건부 엔트로피입니다.

**등가 불순도 감소 형태 (Equivalent impurity-drop form)**:
엔트로피($H$) 외의 다른 불순도 측정 함수 $I$ (예: 지니 불순도)를 사용할 경우, 최적의 분할 $s^*$를 선택하는 일반화된 수식은 다음과 같습니다.

$$
s^* = \arg \max_s \Delta I(s) = \arg \max_s \left[I(\hat{p}_{parent}) - \sum_{z \in \{L, R\}} w_z I(\hat{p}_{Z=z})\right]
$$
*   $\mathbf{\Delta I(s)}$: 분할 $s$에 의해 발생하는 불순도 감소량입니다.
*   $\mathbf{I(\hat{p}_{parent})}$: 분할 전 부모 노드의 불순도입니다.
*   $\mathbf{I(\hat{p}_{Z=z})}$: 자식 노드 $z$의 불순도입니다.
*   **해석**: 이 수식은 부모 노드의 불순도에서 자식 노드들의 가중 평균 불순도를 뺀 값이 가장 커지는 분할을 찾습니다. 엔트로피를 사용하면 위에서 설명한 정보 이득 수식과 동일해집니다.

**구체적 예시**

어떤 대출 신청자가 대출을 받을 수 있는지('승인') 또는 없는지('거절')를 예측하는 의사결정 트리를 만든다고 가정해 봅시다. 현재 노드(부모 노드)에 100명의 신청자 데이터가 있습니다. 이 중 50명은 '승인', 50명은 '거절'입니다.

가능한 분할 기준 중 하나로 '신용 등급이 3등급 이상인가?'를 고려해 봅시다.
*   **분할 전 (부모 노드)**:
    *   '승인': 50명, '거절': 50명
    *   엔트로피 $H_{parent} = - (0.5 \log_2 0.5 + 0.5 \log_2 0.5) = 1$ (가장 높은 불확실성)
*   **분할 후 (자식 노드)**:
    *   **자식 노드 1 (신용 등급 3등급 이상)**: 70명이 이 노드로 이동. 이 중 45명 '승인', 25명 '거절'.
        *   $w_1 = 70/100 = 0.7$
        *   $H_1 = - (45/70 \log_2 45/70 + 25/70 \log_2 25/70) \approx 0.94$
    *   **자식 노드 2 (신용 등급 3등급 미만)**: 30명이 이 노드로 이동. 이 중 5명 '승인', 25명 '거절'.
        *   $w_2 = 30/100 = 0.3$
        *   $H_2 = - (5/30 \log_2 5/30 + 25/30 \log_2 25/30) \approx 0.65$
*   **정보 이득 계산**:
    $IG = H_{parent} - (w_1 H_1 + w_2 H_2)$
    $IG = 1 - (0.7 \times 0.94 + 0.3 \times 0.65)$
    $IG = 1 - (0.658 + 0.195) = 1 - 0.853 = 0.147$

다른 분할 기준(예: '월 소득이 300만원 이상인가?')에 대해서도 같은 방식으로 정보 이득을 계산합니다. 모든 가능한 분할 기준 중에서 정보 이득이 가장 높은 것을 선택하여 현재 노드를 분할합니다. 예를 들어 '월 소득이 300만원 이상인가?'라는 분할이 정보 이득 0.20을 준다면, '월 소득' 특성을 선택하여 분할하게 됩니다.

**시험 포인트**

*   ⭐ **의사결정 트리가 노드를 분할하는 기준**: 정보 이득(Information Gain) 또는 불순도 감소(Impurity Reduction)를 최대화하는 탐욕적(greedy) 방식을 사용한다는 것을 이해해야 합니다.
*   ⭐ **정보 이득 (Information Gain) 수식의 각 항의 의미**:
    *   부모 노드의 엔트로피($H(\hat{p}_{parent})$): 분할 전 불확실성.
    *   자식 노드들의 가중 평균 엔트로피($\sum w_z H(\hat{p}_{Z=z})$): 분할 후 불확실성.
    *   이 둘의 차이가 정보 이득이며, 이 값이 클수록 좋은 분할임을 의미합니다.
*   ⭐ **불순도 함수(Impurity Function)의 종류**: 엔트로피(Entropy)와 지니 불순도(Gini Impurity)가 대표적으로 사용되며, 둘 다 불순도 감소를 측정하는 데 사용될 수 있음을 알아야 합니다.
*   ⭐ **왜 이진 분할을 주로 사용하는가?**: 구현의 용이성, 계산 효율성, 그리고 다중 분할과 비교했을 때 성능상 큰 차이가 없기 때문입니다. (슬라이드에는 명시되지 않았지만 중요 개념)

**핵심 개념**:
*   **연속형 특성 분할 (Splitting Continuous Features)**: 의사결정 트리에서 연속형 값을 가지는 특성($x \in \mathbb{R}$)에 대해 최적의 분할 지점(threshold, $\tau$)을 효율적으로 찾는 방법을 다룹니다. 이는 모든 가능한 실수 값을 분할 지점으로 고려하는 것이 비효율적이기 때문에, 특정 기준에 따라 후보 임계값을 제한하고 탐색하는 기법입니다.
*   **1D 스캔 알고리즘 (1D Scan Algorithm)**: 연속형 특성에 대한 최적의 분할 임계값을 찾기 위한 효율적인 방법입니다. 데이터의 특성 값들을 정렬한 후, 인접한 고유 값들 사이의 중간 지점들만을 후보 임계값으로 고려하여 왼쪽에서 오른쪽으로 한 번 스캔하면서 최적의 분할을 결정합니다.
*   **정보 획득 (Information Gain)**: 분할의 품질을 평가하는 핵심 척도입니다. 부모 노드의 불순도($I(\mathbf{p})$)와 자식 노드들의 가중 평균 불순도($w_L I(\mathbf{p}_L) + w_R I(\mathbf{p}_R)$) 간의 차이를 통해 분할로 인한 불순도 감소량을 측정하며, 이 값이 최대가 되는 분할이 가장 좋습니다.

**코드/수식 해설**:

*   **설정 (Setting)**:
    *   분할하려는 특성(feature) $x$는 실수 공간($\mathbb{R}$)에 속하는 연속형 변수입니다.
    *   레이블(label) $y$는 $K$개의 클래스 중 하나($\{1, \ldots, K\}$)입니다.
    *   주어진 임계값 $\tau$에 따라 데이터를 두 그룹으로 나눕니다:
        *   $L = \{ i : x_i \le \tau \}$ (왼쪽 자식 노드)
        *   $R = \{ i : x_i > \tau \}$ (오른쪽 자식 노드)

*   **1D 스캔 알고리즘 단계**:
    1.  **샘플 정렬**: 모든 샘플을 특성 값 $x$를 기준으로 한 번 정렬합니다. 이 과정의 시간 복잡도는 $O(n \log n)$입니다. 후보 임계값 $\tau$는 연속된 고유 특성 값들 사이의 중간 지점으로 고려합니다.
    2.  **좌우 스캔 (Sweep left-to-right)**: 정렬된 샘플들을 왼쪽에서 오른쪽으로 스캔하면서 $L$과 $R$ 노드의 클래스별 개수를 유지합니다. 이 업데이트는 각 스텝당 상환 시간(amortized time) $O(1)$에 이루어집니다.
    3.  **정보 획득 계산 및 최적 선택**: 각 후보 임계값 $\tau$에 대해 정보 획득 $\Delta I(\tau)$를 계산하고, 이 값이 최대가 되는 $\tau$를 최적의 분할 지점으로 선택합니다.

*   **정보 획득 수식**:
    최적의 $\tau$를 찾기 위해 다음 정보 획득 $\Delta I(\tau)$를 계산하고 이 값이 최대가 되는 $\tau$를 선택합니다.
    $$ \Delta I(\tau) = I(\mathbf{p}) - (w_L I(\mathbf{p}_L) + w_R I(\mathbf{p}_R)) $$
    *   $I(\mathbf{p})$: 분할 전 부모 노드의 불순도(Impurity)입니다. $\mathbf{p}$는 부모 노드의 클래스 비율 분포 벡터입니다.
    *   $I(\mathbf{p}_L)$: 왼쪽 자식 노드의 불순도입니다. $\mathbf{p}_L$은 왼쪽 자식 노드의 클래스 비율 분포 벡터입니다.
    *   $I(\mathbf{p}_R)$: 오른쪽 자식 노드의 불순도입니다. $\mathbf{p}_R$은 오른쪽 자식 노드의 클래스 비율 분포 벡터입니다.
    *   $w_L = |L| / N$: 전체 샘플 수 $N$ 대비 왼쪽 자식 노드에 속하는 샘플 수 $|L|$의 비율입니다.
    *   $w_R = |R| / N$: 전체 샘플 수 $N$ 대비 오른쪽 자식 노드에 속하는 샘플 수 $|R|$의 비율입니다. ($w_L + w_R = 1$이 성립합니다.)
    *   불순도 함수 $I$는 일반적으로 엔트로피(Entropy) 또는 지니 불순도(Gini Impurity)를 사용하며, 이들은 해당 노드의 클래스 비율($p_k$)에만 의존합니다.
        *   **엔트로피 (Entropy)**: $I(\mathbf{p}) = -\sum_{k=1}^{K} p_k \log_2 p_k$
        *   **지니 불순도 (Gini Impurity)**: $I(\mathbf{p}) = 1 - \sum_{k=1}^{K} p_k^2$

**구체적 예시**:
학생들의 통학 거리(연속형 특성)와 선호하는 기숙사 타입(A/B, 레이블) 데이터가 있다고 가정해 봅시다.

| 학생 | 통학 거리($x$) | 기숙사 선호($y$) |
|:----:|:--------------:|:----------------:|
| 1    | 1.2            | A                |
| 2    | 1.8            | A                |
| 3    | 2.5            | B                |
| 4    | 3.0            | A                |
| 5    | 3.2            | B                |
| 6    | 4.0            | B                |

1.  **정렬**: 통학 거리($x$)를 기준으로 데이터를 정렬합니다 (이미 정렬되어 있음).
    *   1.2(A), 1.8(A), 2.5(B), 3.0(A), 3.2(B), 4.0(B)
2.  **후보 임계값 선정**: 인접한 고유 통학 거리 값의 중간 지점을 후보 임계값으로 설정합니다.
    *   $\tau_1 = (1.2+1.8)/2 = 1.5$
    *   $\tau_2 = (1.8+2.5)/2 = 2.15$
    *   $\tau_3 = (2.5+3.0)/2 = 2.75$
    *   $\tau_4 = (3.0+3.2)/2 = 3.1$
    *   $\tau_5 = (3.2+4.0)/2 = 3.6$
3.  **좌우 스캔 & 정보 획득 계산**:
    *   먼저 부모 노드의 불순도를 계산합니다. (총 6명, A:3명, B:3명이라고 가정하고 지니 불순도 사용)
        $I(\mathbf{p}) = 1 - ((3/6)^2 + (3/6)^2) = 0.5$
    *   **$\tau_1 = 1.5$ 일 때**:
        *   $L = \{1.2(\text{A})\}$ (A:1, B:0). $w_L = 1/6$. $I(\mathbf{p}_L) = 0$.
        *   $R = \{1.8(\text{A}), 2.5(\text{B}), 3.0(\text{A}), 3.2(\text{B}), 4.0(\text{B})\}$ (A:2, B:3). $w_R = 5/6$. $I(\mathbf{p}_R) = 1 - ((2/5)^2 + (3/5)^2) = 1 - (4/25 + 9/25) = 12/25 = 0.48$.
        *   $\Delta I(\tau_1) = 0.5 - (1/6 \cdot 0 + 5/6 \cdot 0.48) = 0.5 - 0.4 = 0.1$.
    *   **$\tau_2 = 2.15$ 일 때**:
        *   $L = \{1.2(\text{A}), 1.8(\text{A})\}$ (A:2, B:0). $w_L = 2/6$. $I(\mathbf{p}_L) = 0$.
        *   $R = \{2.5(\text{B}), 3.0(\text{A}), 3.2(\text{B}), 4.0(\text{B})\}$ (A:1, B:3). $w_R = 4/6$. $I(\mathbf{p}_R) = 1 - ((1/4)^2 + (3/4)^2) = 1 - (1/16 + 9/16) = 6/16 = 0.375$.
        *   $\Delta I(\tau_2) = 0.5 - (2/6 \cdot 0 + 4/6 \cdot 0.375) = 0.5 - 0.25 = 0.25$.
    *   이런 식으로 모든 $\tau$ 후보에 대해 $\Delta I(\tau)$를 계산하고 가장 큰 값을 주는 $\tau$를 선택합니다. 이 과정에서 각 스텝마다 $L$과 $R$의 클래스 개수를 효율적으로 업데이트하여 계산량을 줄입니다.

**시험 포인트**:
*   ⭐ 연속형 특성 분할 시 모든 가능한 지점을 탐색하지 않고, **정렬된 고유 값 사이의 중간 지점**만 후보 임계값으로 고려하는 이유를 설명할 수 있어야 합니다. (불순도 지표는 클래스 비율에만 의존하며, 인접한 고유값 사이에서는 분할 결과가 변하지 않기 때문)
*   ⭐ **1D 스캔 알고리즘의 단계** (정렬, 후보 임계값 선정, 좌우 스캔 및 정보 획득 계산)를 순서대로 설명하고 각 단계의 시간 복잡도 ($O(n \log n)$ for sorting, $O(1)$ amortized for sweep)를 이해해야 합니다.
*   ⭐ **정보 획득 $\Delta I(\tau)$ 수식**의 의미를 정확히 이해하고, 구성 요소($I(\mathbf{p})$, $w_L$, $w_R$, $I(\mathbf{p}_L)$, $I(\mathbf{p}_R)$)가 무엇을 나타내는지 설명할 수 있어야 합니다.
*   ⭐ 불순도 함수 $I$로 **엔트로피 또는 지니 불순도**가 사용될 수 있으며, 이들이 클래스 비율에만 의존하는 특성을 가지고 있음을 알고 있어야 합니다.

**핵심 개념**:
의사 결정 트리에서 범주형 특성(Categorical Features)을 사용하여 노드를 분할하는 방법에 대한 내용입니다. 목표는 부모 노드의 불순도(Impurity)를 가장 크게 감소시키는 최적의 분할 기준 $S$를 찾는 것입니다. 범주형 특성 $x$는 $K$개의 고유한 범주 $C = \{c_1, \dots, c_K\}$를 가질 때, 이진 분할(binary split)은 이 범주들 중 특정 부분집합 $S$에 속하는 데이터를 한쪽 노드(좌측 노드 $L$)로, $S$에 속하지 않는 데이터를 다른 노드(우측 노드 $R$)로 분리하는 방식으로 이루어집니다. 범주의 개수 $K$의 크기에 따라 최적의 $S$를 찾는 전략이 달라집니다.

**코드/수식 해설**:
*   **이진 분할 정의**:
    명목형 특성(Nominal feature) $x$가 $K$개의 범주 $C = \{c_1, \dots, c_K\}$를 가질 때, 이진 분할은 범주들의 부분집합 $S \subset C$에 의해 정의됩니다.
    좌측 노드 $L$에 속하는 데이터 포인트 인덱스 집합:
    $L = \{i : x_i \in S\}$
    우측 노드 $R$에 속하는 데이터 포인트 인덱스 집합:
    $R = \{i : x_i \notin S\}$

*   **불순도 감소(Impurity Reduction)**:
    분할의 목표는 불순도 감소 $\Delta I$를 최대화하는 $S$를 선택하는 것입니다.
    $$ \Delta I = I(\mathbf{p}) - (w_L I(\mathbf{p}_L) + w_R I(\mathbf{p}_R)) $$
    여기서:
    *   $I$는 불순도 측정 함수로, 일반적으로 Gini 불순도(Gini impurity) 또는 엔트로피(Entropy)가 사용됩니다.
    *   $\mathbf{p}$는 분할 전 부모 노드의 클래스 비율 분포를 나타냅니다.
    *   $\mathbf{p}_L$과 $\mathbf{p}_R$은 각각 좌측 자식 노드와 우측 자식 노드의 클래스 비율 분포를 나타냅니다.
    *   $w_L$과 $w_R$은 자식 노드의 가중치(child weights)로, 일반적으로 각 자식 노드에 속하는 데이터 포인트의 비율을 사용합니다. 예를 들어, $w_L = |L| / (|L| + |R|)$ 입니다.

*   **Class-1 Rate (이진 분류의 경우)**:
    $K$가 클 때 범주들을 효율적으로 정렬하는 데 사용되는 지표입니다.
    특정 범주 $c$에 대한 Class-1 Rate는 다음과 같이 계산됩니다:
    $$ \text{rate}_1(c) = \mathbb{P}(y = 1 \mid x = c) = \frac{\#\{i : x_i = c, y_i = 1\}}{\#\{i : x_i = c\}} $$
    이는 범주 $c$에 속하는 데이터 포인트 중에서 클래스 1에 해당하는 비율을 의미합니다.

**구체적 예시**:

1.  **K가 작을 때 (Small K)**:
    가정: 범주형 특성 '날씨'가 {'맑음', '흐림', '비'} 세 가지 범주($K=3$)를 가집니다.
    가능한 모든 자명하지 않은 이진 분할 $S$를 시도합니다. 총 $2^{K-1}-1 = 2^{3-1}-1 = 3$가지의 고유한 이진 분할이 존재합니다.
    *   $S = \{\text{맑음}\}$: (맑음) vs (흐림, 비)
    *   $S = \{\text{흐림}\}$: (흐림) vs (맑음, 비)
    *   $S = \{\text{비}\}$: (비) vs (맑음, 흐림)
    각 분할에 대해 불순도 감소 $\Delta I$를 계산하고, $\Delta I$가 최대가 되는 $S$를 최적의 분할로 선택합니다.

2.  **K가 클 때 (Large K)**:
    가정: 범주형 특성 '거주 지역'이 10개 이상의 다양한 도시 이름을 가집니다 ($K \ge 10$). 예를 들어, 특정 상품의 구매 여부(이진 분류)를 예측하는 문제라고 가정합니다.
    이 경우 $2^{K-1}-1$가지의 모든 분할을 탐색하는 것은 계산적으로 비효율적입니다.
    대신, 각 도시에 대한 'Class-1 Rate' (예: 상품 구매율)를 계산합니다.
    예를 들어:
    *   서울: 구매율 75%
    *   부산: 구매율 60%
    *   대구: 구매율 40%
    *   인천: 구매율 65%
    *   ...
    계산된 구매율에 따라 도시들을 정렬합니다:
    {서울 (75%), 인천 (65%), 부산 (60%), ..., 대구 (40%)}
    이제 이 정렬된 순서에 따라 단일 절단(single-cut) 분할을 시도합니다.
    *   $S = \{\text{서울}\}$ vs $\{\text{인천, 부산, 대구, ...}\}$
    *   $S = \{\text{서울, 인천}\}$ vs $\{\text{부산, 대구, ...}\}$
    *   $S = \{\text{서울, 인천, 부산}\}$ vs $\{\text{대구, ...}\}$
    이러한 방식으로 $K-1$가지의 단일 절단 분할만 탐색하여 최적의 분할을 찾습니다. 이는 모든 $2^{K-1}-1$가지 경우를 탐색하는 것보다 훨씬 효율적입니다.

**시험 포인트**:
*   ⭐ **범주형 특성 분할의 정의**: 명목형 특성 $x \in C$에서 이진 분할이 범주의 부분집합 $S \subset C$에 의해 어떻게 정의되는지 정확히 이해하고 설명할 수 있어야 합니다. ($L = \{i : x_i \in S\}$, $R = \{i : x_i \notin S\}$)
*   ⭐ **불순도 감소(Impurity Reduction) 공식**: 분할의 목표인 $\Delta I = I(\mathbf{p}) - (w_L I(\mathbf{p}_L) + w_R I(\mathbf{p}_R))$의 의미와 각 항($I, \mathbf{p}, \mathbf{p}_L, \mathbf{p}_R, w_L, w_R$)의 역할을 설명할 수 있어야 합니다.
*   ⭐ **$K$의 크기에 따른 최적 분할 탐색 전략의 차이**:
    *   **Small K**: 모든 자명하지 않은 $2^{K-1}-1$가지 이진 분할을 시도하는 방법.
    *   **Large K**: `rate1(c)` (Class-1 Rate)와 같은 단일 점수를 사용하여 범주를 정렬하고, 그 순서에 따라 단일 절단(single-cut) 분할을 탐색하는 효율적인 방법.
*   ⭐ `rate1(c)` 공식과 그 의미를 정확히 알아야 하며, 이진 분류 문제에서 특정 클래스(예: 클래스 1)에 속하는 데이터의 비율을 나타낸다는 점을 강조할 수 있어야 합니다.

## 엔트로피(Entropy): 불확실성 정량화

---

### 핵심 개념

정보 엔트로피(Shannon Entropy)는 이산 확률 변수 $Y$의 불확실성 또는 무질서도를 정량적으로 측정하는 척도입니다. 이는 특정 사건의 발생에 대한 '평균 놀라움' 또는 '평균 정보량'으로 해석될 수 있으며, 단위는 비트(bits)입니다. 데이터 분석 및 머신러닝, 특히 의사결정 트리(Decision Trees)와 같은 분류 알고리즘에서 데이터의 불순도(impurity)를 측정하고, 최적의 분할 기준을 찾는 데 중요한 역할을 합니다. 엔트로피 값이 높을수록 불확실성이 크고, 낮을수록 불확실성이 작습니다.

---

### 코드/수식 해설

이산 확률 변수 $Y$의 확률 질량 함수(pmf)가 $p$일 때, 섀넌 엔트로피 $H(Y)$는 다음과 같이 정의됩니다.

$$
H(Y) = -\sum_{y} p(y) \log_2 p(y)
$$

*   **$Y$**: 이산 확률 변수 (예: 동전 던지기 결과, 주사위 눈금)
*   **$p(y)$**: 확률 변수 $Y$가 특정 값 $y$를 가질 확률입니다. (예: 동전 앞면이 나올 확률 0.5)
*   **$\log_2$**: 밑이 2인 로그 함수입니다. 정보량을 비트(bits) 단위로 계산하기 위해 사용됩니다.
*   **$-\sum_{y}$**: 모든 가능한 값 $y$에 대해 $p(y) \log_2 p(y)$ 값을 합산한 후 음수를 취합니다. $0 < p(y) \le 1$ 이므로 $\log_2 p(y)$ 값은 항상 0 또는 음수가 됩니다. 따라서 엔트로피 값을 양수로 만들기 위해 전체 합계에 마이너스를 붙입니다.

이 수식은 각 사건이 발생할 확률 $p(y)$와 그 사건이 가지는 정보량 $(-\log_2 p(y))$을 곱하여 모든 가능한 사건에 대해 평균을 낸 것입니다. 정보량이란 드문 사건일수록 더 많은 정보를 가지고 있다고 봅니다.

**Python 코드 예시**:

```python
import numpy as np

def calculate_entropy(probabilities):
    """
    주어진 확률 분포에 대한 섀넌 엔트로피를 계산합니다.
    p=0인 경우 log2(0)이 -inf가 되므로 해당 항은 0으로 처리합니다.
    """
    entropy = 0
    for p in probabilities:
        if p > 0: # 확률이 0인 경우는 log2(0)이 정의되지 않으므로 무시합니다.
            entropy -= p * np.log2(p)
    return entropy

# 예시 확률 분포
# 1. 공정한 동전: 앞면(0.5), 뒷면(0.5)
p_fair_coin = [0.5, 0.5]
print(f"공정한 동전의 엔트로피: {calculate_entropy(p_fair_coin):.4f} bits")

# 2. 완전히 예측 가능한 동전: 앞면(1.0), 뒷면(0.0)
p_certain_coin = [1.0, 0.0]
print(f"예측 가능한 동전의 엔트로피: {calculate_entropy(p_certain_coin):.4f} bits")

# 3. 공정한 4면 주사위: 각 면 0.25
p_fair_dice = [0.25, 0.25, 0.25, 0.25]
print(f"공정한 4면 주사위의 엔트로피: {calculate_entropy(p_fair_dice):.4f} bits")
```

---

### 구체적 예시

**1. 동전 던지기 예시:**

*   **공정한 동전 (앞면, 뒷면 나올 확률이 각각 0.5)**:
    $p = [0.5, 0.5]$
    $H(Y) = -(0.5 \log_2 0.5 + 0.5 \log_2 0.5)$
    $H(Y) = -(0.5 \times -1 + 0.5 \times -1)$
    $H(Y) = -(-0.5 - 0.5) = -(-1) = 1$ bit
    👉 결과는 앞면 또는 뒷면 중 하나일 수 있으므로 **1비트의 불확실성**을 가집니다. 이는 최대 불확실성 상태입니다.

*   **조작된 동전 (항상 앞면만 나오는 동전, 앞면 확률 1.0, 뒷면 확률 0.0)**:
    $p = [1.0, 0.0]$
    $H(Y) = -(1.0 \log_2 1.0 + 0.0 \log_2 0.0)$
    (참고: $p(y)=0$인 경우 $p(y)\log_2 p(y)$ 항은 0으로 간주합니다)
    $H(Y) = -(1.0 \times 0 + 0)$
    $H(Y) = 0$ bit
    👉 결과가 항상 앞면이라는 것을 확실히 알 수 있으므로 **불확실성은 0**입니다.

**2. 날씨 예측 예시:**

*   **항상 맑은 지역**: 매일 날씨가 맑을 확률이 1이라면 엔트로피는 0입니다. 내일 날씨에 대한 불확실성이 전혀 없기 때문입니다.
*   **사계절이 뚜렷한 지역**: 맑음, 흐림, 비, 눈 등의 날씨가 예측 불가능하게 나타난다면 엔트로피는 높아집니다. 특히 각 날씨의 출현 확률이 비슷하다면 불확실성은 최대가 됩니다.

### 시험 포인트

*   **⭐ 엔트로피의 정의**: 엔트로피는 불확실성, 무질서도, 또는 평균 정보량을 측정하는 척도임을 정확히 이해해야 합니다.
*   **⭐ Shannon Entropy 수식 및 각 항의 의미**:
    $H(Y) = -\sum_{y} p(y) \log_2 p(y)$
    각 $p(y)$, $\log_2$, 그리고 앞에 붙는 음수 기호의 의미를 설명할 수 있어야 합니다. (특히 $\log_2$를 쓰는 이유와 $p(y)=0$일 때의 처리)
*   **⭐ 엔트로피의 주요 특성**:
    *   **범위**: $0 \le H(Y) \le \log_2 K$ ($K$는 가능한 결과의 개수)임을 알아야 합니다.
    *   **최소값 (0)**: 엔트로피는 확률 변수의 결과가 **완전히 예측 가능할 때 (degenerate)** 0이 됩니다. (예: 항상 앞면만 나오는 동전)
    *   **최대값 ($\log_2 K$)**: 엔트로피는 확률 변수의 결과가 **균등 분포(uniform distribution)**를 가질 때 최대가 됩니다. (예: 공정한 동전, 공정한 주사위)
*   **⭐ 머신러닝에서의 활용**: 의사결정 트리와 같은 알고리즘에서 노드의 불순도(impurity)를 측정하는 지표로 사용되며, 정보 이득(Information Gain) 계산의 핵심 요소임을 기억하세요.

**핵심 개념**:
*   **결합 엔트로피 (Joint Entropy, $H(X, Y)$)**: 두 개 이상의 확률 변수 $X, Y$가 동시에 가질 수 있는 불확실성의 양을 측정합니다. 즉, 결합 분포 $p(x,y)$에 대한 예측 불가능성 또는 평균 정보량을 나타냅니다.
*   **엔트로피의 연쇄 법칙 (Chain Rule for Entropy)**: 결합 엔트로피를 단일 변수의 주변 엔트로피(marginal entropy)와 다른 변수가 주어졌을 때의 조건부 엔트로피(conditional entropy)의 합으로 분해하는 규칙입니다. 전체 시스템의 불확실성을 구성 요소별로 나누어 이해하는 데 활용됩니다.
*   **상호 정보량 (Mutual Information, $I(X; Y)$)**: 두 확률 변수 $X$와 $Y$가 서로 공유하는 정보의 양을 측정합니다. 이는 한 변수를 알게 됨으로써 다른 변수에 대한 불확실성이 얼마나 감소하는지를 나타내며, 두 변수 간의 통계적 의존성 정도를 의미합니다. 상호 정보량은 항상 0보다 크거나 같습니다.

**코드/수식 해설**:

1.  **결합 엔트로피 (Joint Entropy)**:
    두 이산 확률 변수 $X$와 $Y$의 결합 분포 $p(x,y)$에 대한 결합 엔트로피는 다음과 같이 정의됩니다.
    $$ H(X, Y) = - \sum_{x} \sum_{y} p(x,y) \log_2 p(x,y) $$
    *   $p(x,y)$: 변수 $X$가 값 $x$를 가지고 동시에 변수 $Y$가 값 $y$를 가질 결합 확률입니다.
    *   $\sum_{x} \sum_{y}$: 모든 가능한 $(x,y)$ 쌍에 대해 합산합니다.
    *   $\log_2$: 밑이 2인 로그를 사용하여 정보의 단위를 비트(bit)로 측정합니다.
    *   이 수식은 단일 변수의 엔트로피 정의를 두 변수의 결합된 경우로 확장한 것으로, 두 변수의 동시 발생에 대한 전체적인 불확실성을 나타냅니다.

2.  **엔트로피의 연쇄 법칙 (Chain Rule for Entropy)**:
    결합 엔트로피 $H(X, Y)$는 다음과 같이 주변 엔트로피와 조건부 엔트로피의 합으로 분해될 수 있습니다.
    $$ H(X, Y) = H(X) + H(Y | X) = H(Y) + H(X | Y) $$
    *   $H(X)$: 변수 $X$의 주변 엔트로피입니다.
    *   $H(Y | X)$: 변수 $X$가 주어졌을 때 변수 $Y$의 조건부 엔트로피입니다. 이는 $X$의 값을 안 상태에서 $Y$에 남아있는 평균적인 불확실성입니다.
    *   이 법칙은 전체 시스템의 불확실성($H(X,Y)$)을 한 변수의 불확실성($H(X)$ 또는 $H(Y)$)과 그 변수가 주어진 조건 하에서 다른 변수의 남은 불확실성($H(Y|X)$ 또는 $H(X|Y)$)으로 나누어 계산할 수 있음을 보여줍니다.

3.  **상호 정보량 (Mutual Information)**:
    두 변수 $X$와 $Y$ 사이의 상호 정보량 $I(X; Y)$는 한 변수를 관찰함으로써 다른 변수에 대해 얻는 정보의 양을 나타냅니다.
    $$ I(X; Y) = H(Y) - H(Y | X) = H(X) - H(X | Y) $$
    또한, 결합 엔트로피를 사용하여 다음과 같이 표현할 수 있습니다.
    $$ I(X; Y) = H(X) + H(Y) - H(X, Y) \geq 0 $$
    *   $H(Y) - H(Y | X)$: 변수 $Y$의 총 불확실성($H(Y)$)에서 변수 $X$가 주어졌을 때의 $Y$의 불확실성($H(Y|X)$)을 뺀 값입니다. 이 차이가 $X$가 $Y$에 대해 제공하는 정보의 양입니다.
    *   $H(X) + H(Y) - H(X, Y)$: 두 변수가 독립이라면 $H(X,Y) = H(X) + H(Y)$이므로, 이 경우 $I(X;Y)=0$이 됩니다. 두 변수가 공유하는 정보가 많을수록 $I(X;Y)$ 값은 커집니다.
    *   상호 정보량은 항상 0보다 크거나 같습니다.

**구체적 예시**:

*   **결합 엔트로피**:
    어떤 지역의 기온(낮음, 보통, 높음)과 습도(낮음, 보통, 높음) 두 변수를 고려해 봅시다. 결합 엔트로피 $H(\text{기온}, \text{습도})$는 이 지역의 날씨 상태(예: (보통 기온, 높은 습도))에 대한 전체적인 불확실성을 측정합니다. 가능한 모든 (기온, 습도) 조합에 대한 확률을 기반으로 이 상태를 예측하는 데 필요한 평균 비트 수를 나타냅니다.

*   **연쇄 법칙**:
    학생의 시험 합격 여부($X$)와 특정 과목의 시험 점수($Y$)를 생각해 봅시다.
    $H(X,Y)$는 학생의 합격 여부와 과목 점수라는 두 정보에 대한 총 불확실성입니다.
    연쇄 법칙 $H(X, Y) = H(X) + H(Y | X)$에 따르면, "학생의 합격 여부에 대한 불확실성($H(X)$)"에 "학생이 합격했다는 것을 알았을 때 과목 점수에 대한 불확실성($H(Y|X)$)"을 더한 것과 같습니다. 예를 들어, 합격한 학생은 특정 점수 이상을 받았을 가능성이 높아 점수에 대한 불확실성 $H(Y|X)$가 감소할 것입니다.

*   **상호 정보량**:
    머신러닝에서 특정 환자의 질병 진단 결과($Y$)와 혈액 검사 수치($X$) 간의 관계를 분석할 때 상호 정보량을 사용할 수 있습니다.
    $I(X; Y) = H(Y) - H(Y | X)$는 "혈액 검사 수치 $X$를 알게 됨으로써 질병 진단 결과 $Y$에 대한 불확실성이 얼마나 감소하는가?"를 나타냅니다. 만약 특정 혈액 수치가 질병 진단과 강하게 연관되어 있다면, $X$와 $Y$ 사이의 상호 정보량이 높을 것이고, 이는 해당 혈액 검사 수치가 질병 진단의 중요한 특성(feature)임을 의미합니다. 특성 선택 과정에서 예측력이 높은 특성을 고르기 위해 활용될 수 있습니다.

**시험 포인트**:
*   ⭐ **결합 엔트로피, 연쇄 법칙, 상호 정보량의 정의 수식**을 정확히 이해하고 암기해야 합니다.
*   ⭐ 각 개념이 **무엇을 측정하는지** (불확실성, 정보 공유량 등) 명확하게 설명할 수 있어야 합니다.
*   ⭐ 특히 **상호 정보량 $I(X;Y)$가 $H(Y) - H(Y|X)$ 또는 $H(X) + H(Y) - H(X,Y)$로 표현될 수 있음**을 이해하고, 왜 항상 0보다 크거나 같은지 설명할 수 있어야 합니다.
*   ⭐ **연쇄 법칙이 조건부 엔트로피와 주변 엔트로피의 관계를 어떻게 설명하는지**에 대한 이해가 중요합니다.
*   ⭐ **머신러닝에서 상호 정보량이 어떻게 활용될 수 있는지** (예: 특성 선택(feature selection)의 척도)에 대한 적용 사례를 이해하는 것이 중요합니다.

**핵심 개념**
조건부 엔트로피(Conditional Entropy) $H(Y | X)$는 확률 변수 $X$의 값을 관측한 후, 확률 변수 $Y$에 남아있는 불확실성의 평균적인 양을 측정하는 척도입니다. 즉, $X$에 대한 정보가 주어졌을 때 $Y$를 예측하는 데 필요한 평균적인 정보량을 나타냅니다. 이는 $X$가 관측되기 전의 $Y$에 대한 불확실성($H(Y)$)이 $X$ 관측을 통해 얼마나 감소하는지(정보 이득, Information Gain)를 파악하는 데 사용되는 핵심 개념입니다.

**코드/수식 해설**

조건부 엔트로피는 다음과 같은 수식으로 정의됩니다.

$$H(Y | X) = \sum_x p(x) H(Y | X = x)$$
$$= -\sum_{x,y} p(x,y) \log_2 p(y | x)$$

*   $H(Y | X)$: $X$가 주어졌을 때 $Y$의 조건부 엔트로피입니다.
*   $\sum_x p(x) H(Y | X = x)$: 이 표현은 $X$의 각 가능한 값 $x$에 대해, $X$가 $x$일 때의 $Y$의 엔트로피 $H(Y | X = x)$를 $X$가 $x$일 확률 $p(x)$로 가중 평균한 값입니다. 즉, $X$의 모든 가능한 값에 대한 $Y$의 불확실성을 평균 낸 것입니다.
*   $H(Y | X = x) = -\sum_y p(y | x) \log_2 p(y | x)$: $X$가 특정 값 $x$일 때의 $Y$의 엔트로피를 나타냅니다. 여기서 $p(y | x)$는 $X=x$라는 조건 하에 $Y=y$일 조건부 확률입니다.
*   $p(x,y)$: $X=x$이면서 $Y=y$일 결합 확률(joint probability)입니다.
*   $p(y | x)$: $X=x$일 때 $Y=y$일 조건부 확률(conditional probability)입니다. 이는 베이즈 정리 등을 통해 $p(x,y) / p(x)$로도 표현될 수 있습니다.
*   $\log_2$: 로그의 밑이 2인 것은 정보량의 단위가 '비트(bits)'임을 의미합니다.

이 수식은 $X$라는 정보를 얻은 후 $Y$에 대한 남은 불확실성의 기댓값(expected residual uncertainty)을 계산하는 방법을 보여줍니다.

**구체적 예시**

**의사결정 트리(Decision Tree)의 특성 선택**

의사결정 트리가 데이터를 분할할 때, 어떤 특성(feature, $X$)을 기준으로 분할해야 가장 '순수'한 자식 노드(즉, $Y$의 불확실성이 낮은 노드)를 얻을 수 있을까요? 이때 조건부 엔트로피가 활용됩니다.

예를 들어, "학생의 시험 합격 여부($Y$)"를 예측하는 의사결정 트리를 만든다고 가정해 봅시다. 그리고 특성으로 "공부 시간($X_1$)"과 "과외 여부($X_2$)"가 있습니다.

1.  **공부 시간($X_1$)으로 분할했을 때의 조건부 엔트로피 $H(Y | X_1)$ 계산**:
    *   공부 시간이 '많음'인 학생들 그룹에서 합격/불합격 비율을 보고 $H(Y | X_1 = \text{많음})$을 계산합니다.
    *   공부 시간이 '적음'인 학생들 그룹에서 합격/불합격 비율을 보고 $H(Y | X_1 = \text{적음})$을 계산합니다.
    *   이 두 값을 각 그룹의 비율($p(X_1=\text{많음})$, $p(X_1=\text{적음})$)로 가중 평균하여 $H(Y | X_1)$을 구합니다.

2.  **과외 여부($X_2$)로 분할했을 때의 조건부 엔트로피 $H(Y | X_2)$ 계산**:
    *   과외를 '함'인 학생들 그룹에서 합격/불합격 비율을 보고 $H(Y | X_2 = \text{함})$을 계산합니다.
    *   과외를 '안함'인 학생들 그룹에서 합격/불합격 비율을 보고 $H(Y | X_2 = \text{안함})$을 계산합니다.
    *   이 두 값을 각 그룹의 비율($p(X_2=\text{함})$, $p(X_2=\text{안함})$)로 가중 평균하여 $H(Y | X_2)$를 구합니다.

만약 $H(Y | X_1)$이 $H(Y | X_2)$보다 작다면, "공부 시간"이라는 특성이 "과외 여부"보다 "합격 여부"에 대한 불확실성을 더 많이 감소시킨다는 의미입니다. 따라서 의사결정 트리는 보통 불확실성을 가장 많이 줄여주는(즉, 조건부 엔트로피를 가장 작게 만드는) 특성을 선택하여 노드를 분할합니다. 이는 정보 이득(Information Gain)을 최대화하는 것과 동일합니다.

**시험 포인트**

*   ⭐ **조건부 엔트로피의 정의와 의미**: $X$를 관측한 후에 $Y$에 남아있는 평균적인 불확실성(잔여 불확실성)이라는 핵심 의미를 정확히 이해하고 설명할 수 있어야 합니다.
*   ⭐ **공식의 이해 및 적용**: 조건부 엔트로피 공식 $H(Y | X) = -\sum_{x,y} p(x,y) \log_2 p(y | x)$ 를 정확히 알고, 각 항($p(x,y)$, $p(y|x)$ 등)이 무엇을 의미하는지 설명할 수 있어야 합니다. 간단한 확률 분포가 주어졌을 때 직접 계산할 수 있어야 합니다.
*   ⭐ **정보 이득과의 관계**: 조건부 엔트로피가 정보 이득(Information Gain)을 계산하는 핵심 구성 요소임을 이해해야 합니다. 정보 이득은 $IG(Y, X) = H(Y) - H(Y | X)$로 정의되며, 이는 $X$라는 정보를 얻음으로써 $Y$의 불확실성이 얼마나 감소하는지를 나타냅니다.
*   ⭐ **머신러닝에서의 활용**: 의사결정 트리와 같은 머신러닝 알고리즘에서 특성(feature)을 선택하거나 분할 기준을 결정하는 데 조건부 엔트로피가 어떻게 사용되는지 예시를 들어 설명할 수 있어야 합니다.

**핵심 개념**:
*   **정보 획득 (Information Gain, IG)**: 의사결정 트리(Decision Tree)에서 노드를 분할할 때, 분할 전후의 불순도(Impurity) 감소량을 측정하는 지표입니다. 불순도 감소량이 클수록 해당 분할이 분류에 더 유용하다고 판단합니다. 즉, 자식 노드들이 부모 노드보다 더 "순수"해지는 정도를 나타냅니다.
*   **이진 분할(Binary Splits)**: 특정 특성($X_j$)에 대한 임계값($\tau$)을 기준으로 데이터를 두 개의 자식 노드(L: Left, R: Right)로 나누는 방법입니다. 예를 들어, $X_j \le \tau$인 경우는 L, $X_j > \tau$인 경우는 R로 분류합니다.
*   **엔트로피 (Entropy, $H$)**: 정보 이론에서 불확실성 또는 불순도를 측정하는 척도입니다. 엔트로피가 높을수록 데이터의 클래스 분포가 균일하여 불순도가 높고, 엔트로피가 낮을수록 특정 클래스로 데이터가 몰려 불순도가 낮습니다. 여기서는 타겟 변수 $Y$의 클래스 확률 분포 $\hat{p}$에 대한 엔트로피를 의미합니다.
*   **가중치($w_z$)**: 각 자식 노드에 속하는 데이터의 비율을 나타냅니다. 이는 전체 노드에 대한 자식 노드의 상대적 중요도를 반영하여 불순도 계산에 사용됩니다.

**코드/수식 해설**:
이 슬라이드에는 직접적인 파이썬 코드는 없지만, 개념을 설명하는 데 필요한 수식들이 있습니다.

1.  **이진 분할 지시자 (Indicator)**:
    데이터 포인트 $X_j$가 임계값 $\tau$보다 작거나 같은 경우 왼쪽 자식 $L$로, 그렇지 않은 경우 오른쪽 자식 $R$로 할당됨을 나타냅니다.
    $$Z = \mathbb{I}[X_j \le \tau] \in \{L, R\}$$
    여기서 $\mathbb{I}[\cdot]$는 조건이 참이면 1, 거짓이면 0을 반환하는 지시 함수이지만, 여기서는 $L$ 또는 $R$을 반환하는 것으로 해석됩니다.

2.  **자식 노드 가중치**:
    각 자식 노드 $z \in \{L, R\}$의 가중치 $w_z$는 해당 자식 노드의 샘플 수 $n_z$를 부모 노드의 전체 샘플 수 $n$으로 나눈 값입니다.
    $$w_L = \frac{n_L}{n}, \quad w_R = \frac{n_R}{n}$$

3.  **정보 획득 (Information Gain) 수식 - 이진 분할**:
    정보 획득은 부모 노드의 엔트로피에서 자식 노드들의 가중 평균 엔트로피를 뺀 값으로 정의됩니다. 이는 분할을 통해 얻어지는 불확실성 감소량을 나타냅니다.
    $$IG = H(Y; \hat{p}_{\text{parent}}) - H(Y | Z)$$
    $$IG = H(Y; \hat{p}_{\text{parent}}) - \sum_{z \in \{L, R\}} w_z H(Y; \hat{p}_{Z=z})$$
    *   $H(Y; \hat{p}_{\text{parent}})$: 분할 전 부모 노드의 타겟 변수 $Y$에 대한 엔트로피. $\hat{p}_{\text{parent}}$는 부모 노드에서의 $Y$의 클래스 확률 분포입니다.
    *   $H(Y | Z)$: $Z$로 분할했을 때의 $Y$의 조건부 엔트로피. 이는 각 자식 노드 $z$의 엔트로피 $H(Y; \hat{p}_{Z=z})$를 해당 자식 노드의 가중치 $w_z$로 평균한 값입니다. $\hat{p}_{Z=z}$는 자식 노드 $z$에서의 $Y$의 클래스 확률 분포입니다.

4.  **정보 획득 (Information Gain) 수식 - 다중 분할 (Multiway)**:
    이진 분할의 개념을 일반화하여 $C$개의 자식 노드로 분할하는 경우입니다.
    $$IG = H(Y; \hat{p}_{\text{parent}}) - \sum_{c=1}^{C} w_c H(Y; \hat{p}_{Z=c})$$
    여기서 $w_c$는 $c$-번째 자식 노드의 가중치이고, $\hat{p}_{Z=c}$는 $c$-번째 자식 노드의 클래스 확률 분포입니다.

**구체적 예시**:
온라인 쇼핑몰에서 고객을 대상으로 마케팅 캠페인을 기획한다고 가정해 봅시다. 고객 데이터를 "구매 여부"($Y$) 기준으로 분류하는 의사결정 트리를 만든다고 할 때:

*   **부모 노드**: 전체 고객 (예: 100명 중 50명이 구매, 50명이 미구매). 이 경우 엔트로피는 매우 높습니다 (최대 엔트로피).
*   **특성 선택**: 고객의 "나이"($X_j$) 특성을 사용하여 "30세 이하"와 "30세 초과"로 고객을 나눈다고 가정해봅시다 ($\tau=30$).
    *   **왼쪽 자식 (L)**: 30세 이하 고객 (예: 60명 중 45명이 구매, 15명이 미구매). 이 노드의 구매율은 75%로, 부모 노드보다 구매/미구매의 불균형이 커져 엔트로피가 낮아집니다.
    *   **오른쪽 자식 (R)**: 30세 초과 고객 (예: 40명 중 5명이 구매, 35명이 미구매). 이 노드의 구매율은 12.5%로, 역시 부모 노드보다 엔트로피가 낮아집니다.
*   **정보 획득 계산**: 분할 전 부모 노드의 엔트로피에서 각 자식 노드의 가중 평균 엔트로피를 뺀 값이 정보 획득이 됩니다. 만약 이 값이 크다면, "나이 30세" 기준으로 나누는 것이 고객의 구매 여부를 예측하는 데 매우 유용하다는 의미가 됩니다.

```python
import numpy as np

def entropy(probabilities):
    """
    주어진 확률 분포에 대한 엔트로피를 계산합니다.
    """
    # 0인 확률 값은 log2(0)이 -inf가 되므로 무시합니다.
    # np.sum()은 제너레이터 표현식과 함께 사용될 때 리스트로 변환하여 성능 저하를 방지합니다.
    return -np.sum([p * np.log2(p) for p in probabilities if p > 0])

# 예시 데이터 (구매/미구매)
# 부모 노드: 총 100명, 구매 50명, 미구매 50명
parent_n_buy = 50
parent_n_nobuy = 50
parent_total = parent_n_buy + parent_n_nobuy
p_parent_buy = parent_n_buy / parent_total
p_parent_nobuy = parent_n_nobuy / parent_total
H_parent = entropy([p_parent_buy, p_parent_nobuy])
print(f"부모 노드 엔트로피: {H_parent:.4f} (불확실성 최대)")

# 자식 노드 L (30세 이하): 총 60명, 구매 45명, 미구매 15명
child_L_n_buy = 45
child_L_n_nobuy = 15
child_L_total = child_L_n_buy + child_L_n_nobuy
p_L_buy = child_L_n_buy / child_L_total
p_L_nobuy = child_L_n_nobuy / child_L_total
H_L = entropy([p_L_buy, p_L_nobuy])
w_L = child_L_total / parent_total
print(f"자식 L 노드 엔트로피: {H_L:.4f}, 가중치 w_L: {w_L:.2f}")

# 자식 노드 R (30세 초과): 총 40명, 구매 5명, 미구매 35명
child_R_n_buy = 5
child_R_n_nobuy = 35
child_R_total = child_R_n_buy + child_R_n_nobuy
p_R_buy = child_R_n_buy / child_R_total
p_R_nobuy = child_R_n_nobuy / child_R_total
H_R = entropy([p_R_buy, p_R_nobuy])
w_R = child_R_total / parent_total
print(f"자식 R 노드 엔트로피: {H_R:.4f}, 가중치 w_R: {w_R:.2f}")

# 분할 후 평균 엔트로피 (조건부 엔트로피)
H_children_avg = (w_L * H_L) + (w_R * H_R)
print(f"자식 노드들의 가중 평균 엔트로피 (H(Y|Z)): {H_children_avg:.4f}")

# 정보 획득 (Information Gain) 계산
IG = H_parent - H_children_avg
print(f"정보 획득 (Information Gain): {IG:.4f} (불확실성 감소량)")
```

**시험 포인트**:
*   ⭐**정보 획득 (Information Gain)의 정의와 목적**을 정확히 이해하고 설명할 수 있어야 합니다. (분할 전후 불확실성/불순도 감소량)
*   ⭐**정보 획득 수식을 이해하고 각 항이 의미하는 바**를 설명할 수 있어야 합니다. 특히, $H(Y; \hat{p}_{\text{parent}})$ (부모 노드 엔트로피)와 $\sum w_z H(Y; \hat{p}_{Z=z})$ (자식 노드들의 가중 평균 엔트로피)의 의미를 구분하는 것이 중요합니다.
*   ⭐**엔트로피의 개념**과 그것이 불순도(Impurity)를 측정하는 데 어떻게 사용되는지 알아야 합니다. (클래스 분포의 균일성)
*   **이진 분할과 다중 분할 간의 IG 수식 차이점**을 이해해야 합니다.
*   **"Bias caution"**: 일반적인 정보 획득이 '많은 가지로 분할하는 것'(high-arity splits)에 유리하게 편향될 수 있다는 점과 이를 해결하기 위한 '정보 획득 비율(Gain Ratio)'의 필요성 (슬라이드에는 언급만 되어있지만 중요한 개념입니다)을 간략하게 설명할 수 있으면 좋습니다.

**핵심 개념**:
의사결정 트리(Decision Tree)는 데이터를 재귀적으로 분할하여 분류 또는 회귀 모델을 생성합니다. 이 슬라이드는 의사결정 트리가 하나의 부모 노드(Parent Node)를 이진 분할(Binary Split)하여 두 개의 자식 노드(Child Node)를 생성하는 초기 과정을 설명합니다. 이 과정에서는 각 노드의 클래스별 샘플 개수, 클래스별 비율, 그리고 해당 자식 노드가 부모 노드에서 차지하는 가중치(Weight)를 계산합니다. 이러한 값들은 트리의 분할 기준(예: 불순도 감소량)을 평가하여 최적의 분할 지점을 찾기 위해 사용됩니다.

**코드/수식 해설**:
*   **부모 노드($N_{parent}$)**:
    *   총 샘플 수: $n_{parent} = 10$
    *   클래스 구성: `orange(O)` 6개, `lemon(L)` 4개
    *   **클래스별 개수**: $(\#O, \#L) = (6, 4)$
    *   **클래스별 비율($\hat{p}_{parent}$)**: 각 클래스 개수를 총 샘플 수로 나눈 값입니다.
        $$ \hat{p}_{parent} = \left( \frac{\#O}{n_{parent}}, \frac{\#L}{n_{parent}} \right) = \left( \frac{6}{10}, \frac{4}{10} \right) = (0.6, 0.4) $$
    *   **가중치**: 부모 노드 자체의 가중치는 1.0으로 간주됩니다.

*   **후보 분할(Candidate Split)**:
    *   부모 노드가 두 개의 자식 노드(`Left`와 `Right`)로 분할됩니다.
    *   왼쪽 자식 노드($N_{Left}$)의 샘플 수: $n_L = 5$
    *   오른쪽 자식 노드($N_{Right}$)의 샘플 수: $n_R = 5$

*   **왼쪽 자식 노드($N_{Left}$, Z = L)**:
    *   클래스 구성: `orange(O)` 4개, `lemon(L)` 1개
    *   **클래스별 개수**: $(4, 1)$
    *   **클래스별 비율($\hat{p}_{Z=L}$)**:
        $$ \hat{p}_{Z=L} = \left( \frac{4}{n_L}, \frac{1}{n_L} \right) = \left( \frac{4}{5}, \frac{1}{5} \right) = (0.8, 0.2) $$
    *   **가중치($w_L$)**: 부모 노드의 샘플 중 왼쪽 자식 노드로 이동한 샘플의 비율입니다.
        $$ w_L = \frac{n_L}{n_{parent}} = \frac{5}{10} = 0.5 $$

*   **오른쪽 자식 노드($N_{Right}$, Z = R)**:
    *   클래스 구성: `orange(O)` 2개, `lemon(L)` 3개
    *   **클래스별 개수**: $(2, 3)$
    *   **클래스별 비율($\hat{p}_{Z=R}$)**:
        $$ \hat{p}_{Z=R} = \left( \frac{2}{n_R}, \frac{3}{n_R} \right) = \left( \frac{2}{5}, \frac{3}{5} \right) = (0.4, 0.6) $$
    *   **가중치($w_R$)**: 부모 노드의 샘플 중 오른쪽 자식 노드로 이동한 샘플의 비율입니다.
        $$ w_R = \frac{n_R}{n_{parent}} = \frac{5}{10} = 0.5 $$

이러한 비율과 가중치는 노드의 불순도(Impurity)를 계산하고, 분할 전후의 불순도 감소량(예: 정보 이득 또는 Gini 불순도 감소)을 통해 해당 분할이 얼마나 효과적인지 평가하는 데 사용됩니다.

**구체적 예시**:
당신이 특정 조건(예: 당도)에 따라 과일을 분류하는 기계를 만들고 있다고 가정해 봅시다. 당신에게는 오렌지(O)와 레몬(L)이 섞여 있는 큰 바구니(부모 노드)가 있습니다. 이 바구니에는 총 10개의 과일이 들어있고, 그중 오렌지가 6개, 레몬이 4개입니다.
당신은 이 바구니를 당도 기준으로 '달콤한 과일'(왼쪽 자식 노드)과 '덜 달콤한 과일'(오른쪽 자식 노드) 두 개의 작은 바구니로 나누려 합니다.

1.  **원래 큰 바구니 (부모 노드)**: 오렌지 6개, 레몬 4개. 오렌지 비율 60%, 레몬 비율 40%.
2.  **'달콤한 과일' 바구니 (왼쪽 자식 노드)**: 5개의 과일이 이 바구니로 들어갔습니다. 이 중 오렌지가 4개, 레몬이 1개입니다. 이 바구니의 과일 중 오렌지 비율은 80%, 레몬 비율은 20%입니다. 이 바구니는 원래 큰 바구니에 있던 과일의 50%를 담고 있습니다.
3.  **'덜 달콤한 과일' 바구니 (오른쪽 자식 노드)**: 나머지 5개의 과일이 이 바구니로 들어갔습니다. 이 중 오렌지가 2개, 레몬이 3개입니다. 이 바구니의 과일 중 오렌지 비율은 40%, 레몬 비율은 60%입니다. 이 바구니 역시 원래 큰 바구니에 있던 과일의 50%를 담고 있습니다.

이 과정을 통해 각 바구니(노드)의 과일 구성 비율을 파악하고, 각 작은 바구니가 전체에서 차지하는 비중(가중치)을 알 수 있습니다. 이 정보로 '당도' 기준이 과일을 얼마나 잘 분리했는지 평가할 수 있습니다.

**시험 포인트**:
*   ⭐ **부모 노드 및 자식 노드의 클래스별 비율 계산**: 주어진 클래스별 개수(`Counts`)를 바탕으로 `Proportions`를 정확히 계산할 수 있어야 합니다. (예: $N_{parent}$의 오렌지 비율은 $6/10 = 0.6$)
*   ⭐ **가중치(Weight)의 의미와 계산**: 자식 노드의 가중치($w_L, w_R$)가 부모 노드 데이터 수에 대한 자식 노드 데이터 수의 비율($n_L/n_{parent}, n_R/n_{parent}$)임을 이해하고 계산할 수 있어야 합니다. 이는 불순도 감소량 계산의 핵심 요소입니다.
*   ⭐ **이진 분할 개념**: 의사결정 트리에서 하나의 부모 노드가 두 개의 자식 노드로 분할되는 기본적인 과정과 각 노드의 역할에 대한 이해가 중요합니다.
*   이 슬라이드 이후에 이어질 Gini impurity, Entropy 등 **불순도 지표 계산** 및 **정보 이득(Information Gain) 또는 불순도 감소(Impurity Reduction) 개념**과 연관되어 출제될 가능성이 높습니다. (예: 제시된 데이터를 바탕으로 Gini impurity를 계산하고 정보 이득을 구하는 문제)

**핵심 개념**:
*   **정보 획득량 (Information Gain)**: 의사결정 트리(Decision Tree)에서 노드를 분할할 때, 분할 전후의 불확실성 감소량, 즉 엔트로피(Entropy) 감소량을 나타내는 지표입니다. 정보 획득량이 클수록 해당 특성(feature)이 데이터를 분류하는 데 더 유용하다는 의미입니다.
*   **엔트로피 감소 (Entropy Reduction)**: 엔트로피는 데이터셋의 불순도(impurity) 또는 무질서도(disorder)를 측정하는 척도입니다. 노드를 분할함으로써 각 자식 노드의 엔트로피가 부모 노드보다 낮아지고, 이는 정보가 획득되었음을 의미합니다. 정보 획득량은 부모 노드의 엔트로피에서 자식 노드들의 가중 평균 엔트로피(조건부 엔트로피)를 뺀 값으로 계산됩니다.

**코드/수식 해설**:

슬라이드는 조건부 엔트로피(Conditional Entropy)의 개념을 통해 분할 후의 불확실성을 계산하는 과정을 보여줍니다.

*   **조건부 엔트로피**: 어떤 변수 $Z$ (여기서는 노드의 분할 방향, L 또는 R)가 주어졌을 때 다른 변수 $Y$ (클래스 레이블)의 불확실성을 나타냅니다.
    $$ H(Y | Z) = \sum_{z \in \{L,R\}} P(Z = z) H(Y | Z = z) $$
    *   $P(Z = z)$: 각 자식 노드(L 또는 R)로 데이터가 할당될 확률. 이는 해당 자식 노드의 데이터 수($n_L$ 또는 $n_R$)를 부모 노드의 총 데이터 수($n$)로 나눈 값입니다.
        *   좌측 자식 노드의 가중치: $w_L = \hat{P}(Z=L) = n_L / n$
        *   우측 자식 노드의 가중치: $w_R = \hat{P}(Z=R) = n_R / n$
    *   $H(Y | Z = z)$: $Z=z$일 때 (즉, 특정 자식 노드에 속할 때) 클래스 $Y$의 엔트로피입니다. 이 값은 해당 자식 노드의 클래스 분포를 이용하여 계산됩니다.
        $$ H(Y | Z = z) = - \sum_{k} p_k \log_2(p_k) $$
        여기서 $p_k$는 자식 노드 $z$ 내에서 클래스 $k$의 비율입니다.

*   **정보 획득량 계산**:
    $$ \text{Information Gain}(Y, Z) = H(Y) - H(Y | Z) $$
    *   $H(Y)$: 분할 전 부모 노드의 엔트로피.
    *   $H(Y | Z)$: 분할 후 자식 노드들의 가중 평균 엔트로피 (위의 조건부 엔트로피).

**구체적 예시**:

슬라이드의 다이어그램을 바탕으로 실제 값을 사용하여 정보 획득량을 계산하는 과정을 추적해 보겠습니다.

1.  **부모 노드**:
    *   총 데이터 수 $n=10$. 클래스 O=6개, 클래스 L=4개.
    *   클래스 분포 $\hat{p}_{\text{parent}} = (0.6, 0.4)$.
    *   부모 노드의 엔트로피 $H(\text{Parent})$ 계산:
        $$ H(\text{Parent}) = - (0.6 \log_2 0.6 + 0.4 \log_2 0.4) \approx - (0.6 \times -0.737 + 0.4 \times -1.322) \approx 0.971 $$

2.  **좌측 자식 노드 (L)**:
    *   데이터 수 $n_L=5$. 클래스 O=4개, 클래스 L=1개.
    *   클래스 분포 $\hat{p}_{Z=L} = (0.8, 0.2)$.
    *   좌측 자식 노드의 엔트로피 $H(Y | Z=L)$ 계산:
        $$ H(Y | Z=L) = - (0.8 \log_2 0.8 + 0.2 \log_2 0.2) \approx - (0.8 \times -0.322 + 0.2 \times -2.322) \approx 0.722 $$
    *   좌측 노드의 가중치 $w_L = n_L/n = 5/10 = 0.5$.

3.  **우측 자식 노드 (R)**:
    *   데이터 수 $n_R=5$. 클래스 O=2개, 클래스 L=3개.
    *   클래스 분포 $\hat{p}_{Z=R} = (0.4, 0.6)$.
    *   우측 자식 노드의 엔트로피 $H(Y | Z=R)$ 계산:
        $$ H(Y | Z=R) = - (0.4 \log_2 0.4 + 0.6 \log_2 0.6) \approx - (0.4 \times -1.322 + 0.6 \times -0.737) \approx 0.971 $$
    *   우측 노드의 가중치 $w_R = n_R/n = 5/10 = 0.5$.

4.  **분할 후 조건부 엔트로피 $H(Y|Z)$ 계산**:
    $$ H(Y | Z) = w_L \times H(Y | Z=L) + w_R \times H(Y | Z=R) $$
    $$ H(Y | Z) = 0.5 \times 0.722 + 0.5 \times 0.971 = 0.361 + 0.4855 = 0.8465 $$

5.  **정보 획득량 (Information Gain) 계산**:
    $$ \text{Information Gain} = H(\text{Parent}) - H(Y | Z) $$
    $$ \text{Information Gain} = 0.971 - 0.8465 = 0.1245 $$
    이 값이 양수이므로, 이 분할은 불확실성을 감소시키고 정보 획득이 발생했음을 의미합니다.

**시험 포인트**:
*   ⭐ **정보 획득량의 의미와 목적**: 정보 획득량은 의사결정 트리에서 최적의 분할 기준을 찾는 데 사용되는 핵심 지표임을 이해해야 합니다. 불순도(엔트로피)를 가장 많이 감소시키는 분할을 선택합니다.
*   ⭐ **엔트로피 및 조건부 엔트로피 공식**: 각 노드의 엔트로피와 분할 후의 조건부 엔트로피를 계산하는 공식을 정확히 이해하고 적용할 수 있어야 합니다. 특히 자식 노드들의 엔트로피를 계산할 때 가중치($P(Z=z)$)를 곱하여 합산하는 조건부 엔트로피 공식의 중요성을 알아야 합니다.
*   ⭐ **계산 과정**: 슬라이드와 같은 예시가 주어졌을 때, 각 단계별로 엔트로피, 조건부 엔트로피, 정보 획득량을 계산하는 과정을 직접 수행할 수 있어야 합니다. (로그 계산은 필요 시 계산기 사용 또는 근삿값 제시)

**핵심 개념**:

*   **엔트로피(Entropy)**: 정보 이론에서 데이터 집합의 불확실성 또는 무질서도를 나타내는 척도입니다. 머신러닝, 특히 의사결정 트리에서는 노드의 '불순도(impurity)'를 측정하는 데 사용됩니다. 엔트로피 값이 높을수록 다양한 클래스가 혼재되어 불확실성이 크고, 낮을수록 특정 클래스로 데이터가 잘 정돈되어 순수함을 의미합니다. 이진 분류의 경우, 클래스 비율이 50:50일 때 엔트로피가 1(최대)이고, 100:0 또는 0:100일 때 엔트로피가 0(최소)이 됩니다.
*   **정보 획득(Information Gain, IG)**: 특정 특성(feature)으로 데이터를 분할했을 때 얻어지는 정보량, 즉 불확실성 감소량을 의미합니다. 분할 전 부모 노드의 엔트로피에서 분할 후 자식 노드들의 가중 평균 엔트로피를 뺀 값으로 계산됩니다. 정보 획득 값이 클수록 해당 특성이 데이터를 더 효과적으로 분류하여 불확실성을 크게 줄여주므로, 의사결정 트리에서 최적의 분할 기준(splitting criterion)을 선택하는 데 활용됩니다.

**코드/수식 해설**:

슬라이드는 의사결정 트리에서 한 노드를 특정 특성을 기준으로 두 개의 자식 노드로 분할했을 때의 정보 획득을 계산하는 과정을 보여줍니다. 모든 엔트로피는 이진 분류(두 클래스) 상황에서 계산되었습니다.

1.  **부모 노드의 엔트로피($H(Y; \hat{p}_{\text{parent}})$)**:
    분할 전 부모 노드의 클래스 분포가 0.6과 0.4일 때의 엔트로피입니다.
    $$
    H(Y; \hat{p}_{\text{parent}}) = -(0.6 \log_2 0.6 + 0.4 \log_2 0.4) \\
    = -(-0.44218 - 0.52877) = \mathbf{0.97095 \text{ bits}}
    $$
    $\log_2$는 밑이 2인 로그를 의미하며, 이는 정보의 단위를 비트(bits)로 나타낸다는 것을 의미합니다.

2.  **좌측 자식 노드의 엔트로피($H(Y; \hat{p}_{Z=L})$)**:
    분할 후 좌측 자식 노드의 클래스 분포가 0.8과 0.2일 때의 엔트로피입니다.
    $$
    H(Y; \hat{p}_{Z=L}) = -(0.8 \log_2 0.8 + 0.2 \log_2 0.2) \\
    = -(-0.25754 - 0.46439) = \mathbf{0.72193 \text{ bits}}
    $$
    부모 노드(0.97095)보다 엔트로피가 감소했으므로, 좌측 자식 노드가 더 순수해졌음을 알 수 있습니다.

3.  **우측 자식 노드의 엔트로피($H(Y; \hat{p}_{Z=R})$)**:
    분할 후 우측 자식 노드의 클래스 분포가 0.4와 0.6일 때의 엔트로피입니다.
    $$
    H(Y; \hat{p}_{Z=R}) = -(0.4 \log_2 0.4 + 0.6 \log_2 0.6) \\
    = -(-0.52877 - 0.44218) = \mathbf{0.97095 \text{ bits}}
    $$
    이 노드의 엔트로피는 부모 노드와 동일합니다. 이는 이 분할이 우측 자식 노드의 불확실성을 줄이지 못했음을 의미합니다. (클래스 비율 0.4:0.6과 0.6:0.4는 동일한 엔트로피 값을 가집니다.)

4.  **가중 평균 자식 엔트로피($H(Y|Z)$)**:
    분할된 자식 노드들의 엔트로피를 각 노드의 데이터 비율($w_z$)로 가중 평균한 값입니다. 슬라이드에서는 좌측(L)과 우측(R) 자식 노드의 데이터 비율이 각각 0.5로 동일하다고 가정합니다.
    $$
    H(Y|Z) = \sum_{z \in \{L, R\}} w_z H(Y; \hat{p}_{Z=z}) \\
    = 0.5 \times H(Y; \hat{p}_{Z=L}) + 0.5 \times H(Y; \hat{p}_{Z=R}) \\
    = 0.5 \times 0.72193 + 0.5 \times 0.97095 \\
    = 0.36096 + 0.48548 = \mathbf{0.84644}
    $$
    이는 특정 특성 $Z$에 따라 분류된 후 목표 변수 $Y$의 평균적인 불확실성을 나타냅니다.

5.  **정보 획득(Information Gain, IG)**:
    부모 노드의 엔트로피에서 가중 평균 자식 엔트로피를 뺀 값입니다. 이 값이 클수록 해당 분할이 데이터를 더 잘 분류하는 데 도움이 됩니다.
    $$
    IG = H(Y; \hat{p}_{\text{parent}}) - H(Y|Z) \\
    = 0.97095 - 0.84644 = \mathbf{0.12451 \text{ bits}}
    $$
    해석: 이 분할은 평균적으로 약 0.125 bits의 레이블 불확실성을 제거했습니다.

---
**구체적 예시**:

어떤 고객 데이터에서 "구매 여부"(구매/미구매)를 예측하는 의사결정 트리를 만든다고 상상해 봅시다.
초기 부모 노드에는 전체 고객 데이터가 있으며, "구매" 고객이 60%, "미구매" 고객이 40%라고 가정합니다. 이때 엔트로피는 약 0.971 bits로, 구매 여부에 대한 불확실성이 높은 상태입니다.

이제 "성별"이라는 특성으로 고객을 분할한다고 가정해 봅시다.
*   **좌측 자식 노드 (성별 = 남성)**: 남성 고객 중 "구매"가 80%, "미구매"가 20%로 나타났습니다. 이 노드의 엔트로피는 약 0.722 bits로, 남성 그룹은 구매 여부가 비교적 명확해졌습니다.
*   **우측 자식 노드 (성별 = 여성)**: 여성 고객 중 "구매"가 40%, "미구매"가 60%로 나타났습니다. 이 노드의 엔트로피는 약 0.971 bits로, 여성 그룹은 구매 여부에 대한 불확실성이 부모 노드와 크게 다르지 않습니다.

만약 전체 고객 중 남성 고객과 여성 고객의 비율이 50%씩이었다면,
가중 평균 자식 엔트로피는 $0.5 \times 0.722 + 0.5 \times 0.971 = 0.846$ bits가 됩니다.
이 경우 정보 획득은 $0.971 - 0.846 = 0.125$ bits가 됩니다.

이는 "성별"이라는 특성으로 고객을 분할함으로써, 전체적으로 고객의 "구매 여부"에 대한 불확실성을 평균적으로 0.125 bits만큼 줄일 수 있었다는 의미입니다. 만약 "나이대"나 "거주 지역" 같은 다른 특성으로 분할했을 때 정보 획득이 더 크게 나온다면, 해당 특성이 고객의 구매 여부를 예측하는 데 "성별"보다 더 유용하다고 판단할 수 있습니다.

**시험 포인트**:

*   ⭐ **엔트로피의 정의와 계산 공식**을 정확히 이해하고 있어야 합니다. (특히 이진 분류에서 $-(p \log_2 p + (1-p) \log_2 (1-p))$ 형태)
*   ⭐ **정보 획득(Information Gain)의 개념**: 분할 전 부모 노드의 엔트로피에서 분할 후 자식 노드들의 **가중 평균 엔트로피**를 뺀 값이며, 그 의미(불확실성 감소량)를 설명할 수 있어야 합니다.
*   ⭐ **의사결정 트리에서 정보 획득의 중요성**: 트리의 각 노드에서 최적의 분할 기준을 선택하는 핵심 지표임을 알아야 합니다. 정보 획득을 최대화하는 특성으로 분할하여 트리를 성장시킵니다.
*   가중 평균 엔트로피를 계산할 때, 각 자식 노드의 엔트로피에 해당 노드가 전체 데이터에서 차지하는 **비율(weight)**을 곱해야 함을 놓치지 않도록 주의해야 합니다.
*   엔트로피 값이 낮아질수록 해당 노드가 더 '순수'해지고 분류가 쉬워진다는 것을 해석할 수 있어야 합니다.

### Gini Impurity Reduction

---

-   **핵심 개념**:
    *   **Gini 불순도 (Gini Impurity)**: 결정 트리(Decision Tree)에서 노드의 불순도(Impurity)를 측정하는 지표 중 하나입니다. 한 노드에 여러 클래스의 데이터가 얼마나 섞여 있는지를 나타냅니다. 0에 가까울수록 순수하며(한 클래스의 데이터만 존재), 1에 가까울수록 불순합니다(클래스들이 고루 분포). 결정 트리는 이 지니 불순도를 최소화하는 방향으로 분할을 진행합니다.
    *   **Gini 불순도 감소 (Gini Impurity Reduction 또는 Gini Gain)**: 부모 노드의 지니 불순도에서 자식 노드들의 가중 평균 지니 불순도를 뺀 값입니다. 이 값이 클수록 해당 분할이 데이터를 더 효과적으로 분류하여 노드의 순수도를 높인다는 의미입니다. 결정 트리는 노드를 분할할 때 지니 불순도 감소량이 최대가 되는 특성(feature)과 임계값(threshold)을 선택합니다.

-   **코드/수식 해설**:
    1.  **Gini 불순도 공식**:
        $$ G(\mathbf{p}) = 1 - \sum_{k} p_k^2 $$
        여기서 $p_k$는 해당 노드에서 $k$번째 클래스에 속하는 데이터의 비율을 나타냅니다. 모든 클래스의 비율 제곱을 합한 값을 1에서 빼줍니다.
    2.  **부모 노드의 Gini 불순도**:
        $$ G(\hat{p}_{parent}) = 1 - (0.6^2 + 0.4^2) = 1 - (0.36 + 0.16) = 1 - 0.52 = 0.48 $$
        이는 부모 노드에 클래스 A가 60%, 클래스 B가 40% 있다고 가정했을 때의 불순도입니다.
    3.  **자식 노드들의 Gini 불순도**:
        $$ G(\hat{p}_{Z=L}) = 1 - (0.8^2 + 0.2^2) = 1 - (0.64 + 0.04) = 1 - 0.68 = 0.32 $$
        좌측 자식 노드는 클래스 A가 80%, 클래스 B가 20%로, 부모보다 더 순수해졌습니다.
        $$ G(\hat{p}_{Z=R}) = 1 - (0.4^2 + 0.6^2) = 1 - (0.16 + 0.36) = 1 - 0.52 = 0.48 $$
        우측 자식 노드는 클래스 A가 40%, 클래스 B가 60%로, 불순도 자체는 부모 노드와 동일합니다.
    4.  **분할 후 자식 노드들의 가중 평균 Gini 불순도**:
        $$ G_{children} = \sum_{z \in \{L, R\}} w_z G(\hat{p}_{Z=z}) = 0.5 \times 0.32 + 0.5 \times 0.48 = 0.16 + 0.24 = 0.40 $$
        여기서 $w_z$는 전체 부모 노드 데이터 중 해당 자식 노드($z$)로 가는 데이터의 비율(가중치)입니다. 예시에서는 부모 노드 데이터의 절반이 좌측 자식 노드로, 나머지 절반이 우측 자식 노드로 갔다고 가정하고 있습니다.
    5.  **불순도 감소량 (Impurity Reduction, Gini Gain)**:
        $$ \Delta G = G(\hat{p}_{parent}) - G_{children} = 0.48 - 0.40 = 0.08 $$
        이 값은 해당 분할로 인해 지니 불순도가 0.08만큼 감소했음을 의미합니다.

-   **구체적 예시**:
    어떤 학교에서 학생들을 시험 성적에 따라 'A학점 그룹'과 '비A학점 그룹'으로 나누려고 합니다. 처음에는 전체 학생(부모 노드) 중 60%가 'A학점 그룹', 40%가 '비A학점 그룹'으로 예상됩니다. 이 노드의 지니 불순도는 0.48입니다.
    이제 '중간고사 점수 80점 이상'이라는 기준으로 학생들을 나누었다고 해봅시다.
    *   **좌측 자식 노드 (중간고사 80점 이상)**: 이 그룹은 80%가 'A학점 그룹', 20%가 '비A학점 그룹'이라고 가정하면 지니 불순도는 0.32입니다.
    *   **우측 자식 노드 (중간고사 80점 미만)**: 이 그룹은 40%가 'A학점 그룹', 60%가 '비A학점 그룹'이라고 가정하면 지니 불순도는 0.48입니다.
    만약 전체 학생의 절반이 80점 이상, 절반이 80점 미만이었다면, 분할 후 두 그룹의 평균 불순도는 가중 평균으로 계산되어 0.40이 됩니다.
    결과적으로 이 '중간고사 점수' 기준 분할은 전체 불순도를 $0.48 - 0.40 = 0.08$만큼 감소시켰습니다. 학교는 이처럼 불순도를 가장 많이 감소시키는 기준(예: 중간고사 점수, 출석률 등)을 찾아 학생들을 분류하는 규칙을 만들어갈 수 있습니다.

-   **시험 포인트**:
    *   ⭐ **Gini 불순도 공식**: $G(\mathbf{p}) = 1 - \sum_k p_k^2$를 정확히 이해하고 적용할 수 있어야 합니다. 각 $p_k$가 무엇을 의미하는지 파악하는 것이 중요합니다.
    *   ⭐ **Gini 불순도 감소량 계산 과정**: 부모 노드의 지니 불순도, 각 자식 노드의 지니 불순도, 그리고 자식 노드들의 가중 평균 지니 불순도를 계산하는 과정을 숙지하고, 최종적으로 이 감소량($\Delta G$)을 구하는 방법을 알아야 합니다.
    *   ⭐ **Gini 불순도 감소의 의미**: 이 값이 클수록 좋은 분할임을 이해하고, 결정 트리 학습 과정에서 불순도 감소량이 최대가 되는 분할을 선택한다는 원리를 설명할 수 있어야 합니다.
    *   엔트로피(Entropy)와 정보 이득(Information Gain) 개념과 비교하여 지니 불순도가 어떤 차이점과 공통점을 가지는지 파악하는 것도 중요합니다. (슬라이드 요약에 엔트로피 감소량 언급)

## 데이터분석 입문 (CSED226) - 슬라이드 분석 노트

### 핵심 개념

*   **특징 공학 (Feature Engineering)**: 기존의 원시 특징(예: height, width)을 조합하거나 변형하여 새로운 특징(예: `roundness proxy r = height - width`)을 생성하는 과정입니다. 이는 모델이 데이터의 패턴을 더 잘 학습할 수 있도록 정보를 강화하는 핵심적인 단계입니다.
*   **의사결정 트리 분할 (Decision Tree Splitting)**: 의사결정 트리는 특정 특징($r$)과 임계값($\tau$)을 기준으로 데이터를 두 개의 하위 집합으로 분할합니다. 이 분할은 각 하위 집합의 불순도(impurity)를 최소화하고 정보 이득(Information Gain)을 최대화하는 방향으로 이루어집니다.
*   **엔트로피 ($H$)**: 특정 데이터 집합의 무질서도 또는 불확실성을 측정하는 척도입니다. 엔트로피 값이 낮을수록 해당 집합 내의 데이터는 특정 클래스로 잘 분류되어 있다는 것을 의미합니다.
*   **정보 이득 ($IG$)**: 어떤 특징을 기준으로 데이터를 분할했을 때 얻을 수 있는 불순도(엔트로피)의 감소량을 나타냅니다. 정보 이득이 높을수록 해당 특징으로 분할하는 것이 데이터 분류에 더 효과적임을 의미합니다.
*   **재귀적 분할 (Recursive Splitting)**: 한 번의 분할로 모든 불순도가 제거되지 않으면, 트리는 다른 특징(예: width, height)을 사용하여 하위 집합을 다시 분할하는 과정을 반복합니다. 이 과정을 통해 더욱 순수한 노드(데이터 집합)를 생성하며 트리의 깊이가 깊어집니다.
*   **시각화 아이디어 (Visualization Idea)**: 의사결정 트리의 각 분할은 특징 공간(Feature Space)에서 수직 또는 수평선을 그어 공간을 직사각형 영역으로 나눕니다. 각 직사각형 영역에는 해당 영역에 속하는 데이터의 주된 클래스 레이블이 할당됩니다.

### 코드/수식 해설

*   **Roundness Proxy 정의**:
    특징 공학을 통해 `height`와 `width`라는 두 특징을 사용하여 `r`이라는 새로운 특징을 정의합니다.
    ```
    r = height - width
    ```
*   **분할 조건 (Split Condition)**:
    의사결정 트리가 노드를 분할할 때 사용하는 조건입니다. `r` 값이 특정 임계값($\tau$)보다 작거나 같은지 여부를 확인합니다.
    $$ r \le \tau $$
*   **엔트로피 ($H$) 계산**:
    주어진 데이터 집합 $S$에서 클래스 $c$가 나타날 확률 $p(c)$를 기반으로 엔트로피를 계산합니다.
    $$ H(S) = - \sum_{c \in C} p(c) \log_2 p(c) $$
    여기서 $C$는 모든 클래스의 집합입니다. 슬라이드에 언급된 $\hat{p}$는 $p(c)$의 추정치입니다.
*   **정보 이득 ($IG$) 계산**:
    특징 $A$를 사용하여 데이터 집합 $S$를 분할했을 때 얻는 정보 이득은 분할 전 엔트로피에서 분할 후 엔트로피의 가중 평균을 뺀 값입니다.
    $$ IG(S, A) = H(S) - \sum_{v \in Values(A)} \frac{|S_v|}{|S|} H(S_v) $$
    여기서 $Values(A)$는 특징 $A$가 가질 수 있는 값들의 집합이고, $S_v$는 특징 $A$의 값이 $v$인 $S$의 부분집합입니다.

### 구체적 예시

어떤 물체가 '세로로 긴 직사각형', '가로로 긴 직사각형', '정사각형' 중 어느 것인지 분류하는 문제라고 가정해 봅시다. 입력 특징은 `height`와 `width`입니다.

1.  **특징 공학**: 먼저 `roundness proxy r = height - width`라는 새로운 특징을 만듭니다. `r`이 양수면 세로로 긴 경향이 있고, 음수면 가로로 긴 경향이 있으며, 0에 가까우면 정사각형에 가깝다고 볼 수 있습니다.
2.  **첫 번째 분할**: '가장 정보 이득이 높은' 분할 기준을 찾습니다. 예를 들어, `r <= 0` (즉, `height <= width`)이 가장 좋다고 결정되었다고 해봅시다. 이 분할은 데이터를 "세로가 길거나 정사각형인 물체" 그룹(`r > 0`)과 "가로가 긴 물체" 그룹(`r <= 0`)으로 나눕니다.
    *   각 그룹에서 '세로로 긴', '가로로 긴', '정사각형' 클래스의 개수를 세고($\hat{p}$), 각 그룹의 엔트로피를 계산($H$)합니다.
    *   이 분할이 전체 시스템의 엔트로피를 얼마나 줄였는지 정보 이득($IG$)을 계산하여, 이 분할이 효과적인지 평가합니다.
3.  **재귀적 분할**: 만약 "세로가 길거나 정사각형인 물체" 그룹에서 여전히 '세로로 긴' 물체와 '정사각형' 물체가 섞여 있어 엔트로피가 높다면, 이 그룹 내에서 다시 `height <= 10cm`나 `width <= 5cm`와 같은 다른 특징을 기준으로 추가 분할을 시도하여 엔트로피를 더 줄여나갑니다.
4.  **시각화**: 2차원 특징 공간(x축: width, y축: height)에서 `r <= 0` 분할은 `height = width`라는 직선(기울기 1)을 그어 공간을 나눕니다. 이어서 `height <= 10cm`와 같은 분할은 수평선을, `width <= 5cm`와 같은 분할은 수직선을 그려 공간을 더 작은 직사각형 영역으로 쪼개는 것과 같습니다. 각 직사각형 영역은 최종적으로 하나의 클래스 레이블을 가집니다.

### 시험 포인트

*   ⭐ **특징 공학의 중요성**: 왜 새로운 특징을 만들어야 하는지, 그 목적은 무엇인지 설명할 수 있어야 합니다. (예: `height - width`와 같은 파생 특징이 문제 해결에 어떻게 기여하는가?)
*   ⭐ **엔트로피와 정보 이득의 개념 및 역할**: 의사결정 트리가 분할 기준을 선택할 때 엔트로피와 정보 이득이 어떻게 활용되는지, 각각의 의미를 정확히 이해하고 설명할 수 있어야 합니다. (수식의 의미 파악)
*   ⭐ **의사결정 트리의 분할 과정**: 재귀적으로 분할하며 엔트로피를 감소시키는 과정의 흐름을 이해하고 있어야 합니다.
*   ⭐ **특징 공간 시각화**: 의사결정 트리의 각 분할이 특징 공간에서 어떻게 시각화되는지 (수직/수평선, 직사각형 영역 분할) 이해하고 있어야 합니다.

**핵심 개념**
결정 트리 생성 알고리즘은 데이터를 재귀적으로 분할하여 분류 모델을 구축합니다. 각 분할은 '불순도(impurity)'를 가장 크게 감소시키는 기준으로 선택되며, 트리는 특정 정지 조건(stopping criteria)에 도달할 때까지 성장합니다. 생성된 트리의 과적합(overfitting)을 방지하기 위해 '가지치기(pruning)' 과정이 중요하게 사용됩니다.

**코드/수식 해설**

```
BuildTree(dataset D, depth d):
  if Pure(D) or d >= max_depth or |D| < min_leaf:
    return Leaf( majority_class(D), dist=class_freqs(D) )

  # Local objective: s* = arg max_s ΔI(s)
  # where ΔI(s) = I(parent) - Σ_z∈{L,R} w_z I(child_z)
  # e.g., entropy or gini
  best = argmax_s ImpurityDrop(D, s)

  (DL, DR) = ApplySplit(D, best)
  return Node(split=best,
              left=BuildTree(DL, d+1),
              right=BuildTree(DR, d+1))
```

*   **`BuildTree(dataset D, depth d)`**: 결정 트리를 재귀적으로 구축하는 메인 함수입니다. 현재 노드의 데이터셋 `D`와 현재 트리의 깊이 `d`를 인자로 받습니다.
*   **정지 조건 (Base Cases)**:
    *   `if Pure(D)`: 현재 노드의 모든 데이터 샘플이 동일한 클래스에 속하여 불순도가 0인 경우.
    *   `or d >= max_depth`: 트리의 현재 깊이 `d`가 미리 설정된 최대 깊이 `max_depth`에 도달한 경우.
    *   `or |D| < min_leaf`: 현재 노드의 데이터 샘플 수 `|D|`가 최소 리프 노드 샘플 수 `min_leaf`보다 작은 경우.
    *   이 세 조건 중 하나라도 만족하면 더 이상 분할하지 않고 `Leaf` 노드를 반환합니다. `Leaf` 노드는 해당 노드의 데이터셋 `D`에서 가장 많은 비중을 차지하는 클래스를 예측하고, 클래스별 분포(`class_freqs(D)`)를 저장합니다.
*   **최적 분할 찾기 (Local Objective)**:
    *   `s* = argmax_s ΔI(s)`: 현재 데이터셋 `D`를 분할할 수 있는 모든 가능한 분할 기준 `s` 중에서, 불순도 감소량 `ΔI(s)`를 최대로 만드는 최적의 분할 기준 `s*`를 찾습니다.
    *   **불순도 감소량($\Delta I(s)$) 수식**:
        $$ \Delta I(s) = I(parent) - \sum_{z \in \{L,R\}} w_z I(child_z) $$
        *   $I(parent)$: 분할 전 부모 노드의 불순도입니다.
        *   $I(child_z)$: 분할 후 생성된 왼쪽 자식 노드 (L) 또는 오른쪽 자식 노드 (R)의 불순도입니다.
        *   $w_z$: 각 자식 노드에 할당된 데이터 샘플 수의 비율, 즉 $|child_z| / |parent|$ 입니다.
        *   $I()$의 예시로는 **엔트로피(entropy)** 또는 **지니 불순도(gini impurity)**가 주로 사용됩니다. 이 수식은 분할을 통해 얻는 정보 이득(Information Gain)과 동일한 개념입니다.
    *   `best = argmax_s ImpurityDrop(D, s)`: 데이터셋 `D`에 대해 `ImpurityDrop`을 최대로 하는 분할 `s`를 찾습니다.
*   **재귀적 분할 (Recursive Splitting)**:
    *   `(DL, DR) = ApplySplit(D, best)`: 최적의 분할 기준 `best`를 사용하여 데이터셋 `D`를 두 개의 자식 데이터셋 `DL` (왼쪽)과 `DR` (오른쪽)으로 나눕니다.
    *   `return Node(...)`: 최적의 분할 기준(`best`)을 가진 내부 `Node`를 생성하고, `left`와 `right` 자식 노드는 각각 `DL`과 `DR` 데이터셋에 대해 `BuildTree` 함수를 재귀적으로 호출하여 구축합니다. 이때 깊이는 `d+1`로 증가합니다.

*   **Pruning (가지치기)**:
    *   트리가 훈련 데이터에 너무 잘 맞아 새로운 데이터에 대한 예측 성능이 떨어지는 **과적합(overfitting)**을 방지하기 위한 기법입니다.
    *   **post-prune (사후 가지치기)**: 트리를 완전히 성장시킨 후, 별도의 **검증 세트(validation set)**를 사용하여 트리의 성능이 떨어지지 않는 범위 내에서 불필요한 가지를 잘라냅니다.
    *   **cost-complexity pruning (비용-복잡도 가지치기)**: 트리의 복잡성에 페널티를 부여하는 비용 함수를 사용하여 최적의 가지치기 수준을 찾습니다.

**구체적 예시**
*   **`Pure(D)` 예시**: 어떤 노드에 속한 데이터가 모두 '개'라는 클래스 하나로만 이루어져 있다면, 이 노드는 더 이상 나눌 필요 없이 '개'를 예측하는 리프 노드가 됩니다.
*   **불순도 감소 예시**: 부모 노드에 '남성 50명, 여성 50명'의 데이터가 있다고 가정합시다. 이 노드의 불순도는 높습니다. 여기서 '키 170cm 이상'이라는 기준으로 분할했더니, 왼쪽 자식 노드에 '남성 40명, 여성 10명'이, 오른쪽 자식 노드에 '남성 10명, 여성 40명'이 모였다면, 각 자식 노드는 부모 노드보다 성별에 대한 불순도가 감소했습니다. 이 불순도 감소량을 정량화한 것이 $\Delta I(s)$입니다.
*   **가지치기 예시**: 날씨를 예측하는 트리를 만들 때, '습도 87.3%이고 바람 2.5m/s이며 구름이 낀 시간'과 같은 너무 세부적인 규칙으로 트리가 깊어지면, 훈련 데이터에는 완벽하게 맞더라도 새로운 날씨 데이터에는 일반화되지 않아 예측 오류가 커질 수 있습니다. 가지치기는 이러한 지나치게 구체적인 규칙을 제거하여 모델을 단순화하고 일반화 성능을 향상시킵니다.

**시험 포인트**
*   ⭐ 결정 트리 생성 알고리즘의 **재귀적 특성**을 설명하고, 트리의 성장을 멈추는 **세 가지 주요 정지 조건**(`Pure(D)`, `max_depth`, `min_leaf`)을 정확히 이해해야 합니다.
*   ⭐ **불순도 감소($\Delta I(s)$)의 개념과 그 계산 방법** (부모 불순도 - 자식 불순도의 가중합)을 수식과 함께 설명할 수 있어야 합니다. 이는 정보 이득(Information Gain)의 핵심입니다.
*   ⭐ **엔트로피와 지니 불순도**가 불순도 측정 지표로 사용되는 이유와 각각의 특징을 비교 설명할 수 있어야 합니다.
*   ⭐ **가지치기(Pruning)의 목적**이 무엇인지(과적합 방지) 이해하고, **post-prune** 및 **cost-complexity pruning**과 같은 주요 가지치기 방법들을 구분하여 설명할 수 있어야 합니다.

## Feature Selection in Trees

### 핵심 개념
*   **Local selection (지역 선택)**: 의사결정 트리는 각 노드에서 데이터를 분할할 최적의 피처를 선택합니다. 이 선택 기준은 해당 피처로 분할했을 때 얻을 수 있는 **불순도 감소(Impurity Reduction)**를 최대화하는 것입니다.
*   **Regularization (정규화)**: 트리가 학습 데이터에 과적합되는 것을 방지하기 위한 기법들입니다. 트리의 복잡도를 제한하여 일반화 성능을 높입니다.
*   **Handling missing values (결측값 처리)**: 데이터에 누락된 값이 있을 경우, 트리가 이를 어떻게 처리하고 분할을 진행할지에 대한 전략입니다.
*   **Categoricals (범주형 피처)**: 성별, 지역, 등급 등과 같은 범주형 데이터를 의사결정 트리에서 효과적으로 활용하기 위한 방법입니다.

### 코드/수식 해설
*   **불순도 감소 (Impurity Reduction)**:
    의사결정 트리는 노드를 분할할 때 불순도(Impurity)를 가장 많이 줄이는 피처와 분할점을 선택합니다.
    주로 사용되는 불순도 지표는 엔트로피(Entropy)와 지니 불순도(Gini Impurity)입니다.

    **엔트로피 (Entropy)**:
    정보 이론에서 불확실성의 척도입니다. 엔트로피가 높으면 불확실성이 크고, 낮으면 불확실성이 작습니다.
    $$
    Entropy(S) = -\sum_{i=1}^{c} p_i \log_2(p_i)
    $$
    여기서 $S$는 현재 노드의 데이터셋, $c$는 클래스의 개수, $p_i$는 클래스 $i$에 속하는 샘플의 비율입니다.

    **지니 불순도 (Gini Impurity)**:
    데이터셋에서 무작위로 선택한 두 샘플이 서로 다른 클래스에 속할 확률입니다. 0이면 완벽하게 순수하고, 1에 가까우면 불순합니다.
    $$
    Gini(S) = 1 - \sum_{i=1}^{c} p_i^2
    $$

    **정보 이득 (Information Gain)**:
    분할 전 엔트로피에서 분할 후 자식 노드들의 가중 평균 엔트로피를 뺀 값입니다. 이 값이 클수록 해당 분할이 유용합니다.
    $$
    IG(S, A) = Entropy(S) - \sum_{v \in Values(A)} \frac{|S_v|}{|S|} Entropy(S_v)
    $$
    여기서 $IG(S, A)$는 데이터셋 $S$에 대해 속성 $A$를 사용했을 때의 정보 이득, $Values(A)$는 속성 $A$가 가질 수 있는 값들의 집합, $S_v$는 속성 $A$가 $v$ 값을 가지는 $S$의 부분집합입니다.

*   **정규화 (Regularization) 파라미터 예시 (scikit-learn)**:
    ```python
    from sklearn.tree import DecisionTreeClassifier

    # 의사결정 트리의 복잡도를 제어하는 파라미터들
    model = DecisionTreeClassifier(
        max_depth=5,            # 트리의 최대 깊이 제한
        min_samples_split=20,   # 노드를 분할하기 위한 최소 샘플 수
        min_samples_leaf=10,    # 리프 노드가 가져야 할 최소 샘플 수
        max_features='sqrt'     # 각 노드에서 최적의 분할을 찾기 위해 고려할 피처의 수 (랜덤 서브스페이스)
                                # 'sqrt': 전체 피처 수의 제곱근, 'log2': 로그2, 또는 정수
    )
    ```

### 구체적 예시

*   **Local selection (지역 선택)**:
    어떤 환자의 질병 유무를 예측하는 트리를 만든다고 가정해 봅시다. 현재 노드에 100명의 환자 데이터(50명은 질병 있음, 50명은 질병 없음)가 있습니다.
    1.  '체온 > 37.5도'로 분할: 한쪽은 (질병 40, 건강 10), 다른 쪽은 (질병 10, 건강 40)이 됩니다.
    2.  '기침 유무'로 분할: 한쪽은 (질병 30, 건강 20), 다른 쪽은 (질병 20, 건강 30)이 됩니다.
    이때, '체온 > 37.5도'로 분할했을 때의 불순도 감소(예: 지니 불순도 감소)가 더 크다면, 트리는 해당 분할을 선택하여 다음 노드를 만듭니다.

*   **Regularization (정규화)**:
    쇼핑몰 고객의 구매 이력을 기반으로 다음 달 구매액을 예측하는 트리를 만듭니다.
    *   `max_depth=7`: 트리의 최대 깊이를 7로 제한하여, 트리가 너무 세부적인 규칙(예: "지난 달 방문 횟수 3회이고, 장바구니에 2개 상품을 담았고, 화요일 오전에 접속했으며...")까지 학습하여 과적합되는 것을 방지합니다.
    *   `min_samples_leaf=5`: 리프 노드에 최소 5명 이상의 고객이 포함되도록 합니다. 만약 어떤 분할로 인해 리프 노드에 1~2명만 남게 된다면, 이는 너무 특정 고객의 패턴에 맞춰진 규칙이므로, 이 분할을 허용하지 않습니다.

*   **Handling missing values (결측값 처리)**:
    주택 가격 예측 모델에서 '차고 면적' 피처에 결측값이 있는 주택 데이터가 있습니다.
    *   **Imputation (결측치 대체)**: 트리를 학습하기 전에 '차고 면적'의 결측값을 해당 지역의 평균 '차고 면적'이나, 비슷한 크기의 주택들의 평균 '차고 면적' 등으로 채워 넣습니다.
    *   **Surrogate splits (대리 분할)**: 만약 '차고 면적 > 20평'이 중요한 분할 기준인데 특정 주택의 차고 면적이 누락되어 있다면, 트리는 차선책으로 결측값이 없는 다른 피처(예: '주택 건축 연도 < 2000년')가 '차고 면적 > 20평'과 유사한 방식으로 데이터를 분할하는지 확인하고, 유사도가 높으면 이를 대리 분할로 사용합니다. 이 방법을 사용하면 결측값 때문에 해당 샘플을 버리거나 임의로 채울 필요 없이 트리를 진행할 수 있습니다.

*   **Categoricals (범주형 피처)**:
    사용자의 선호 영화 장르(액션, 코미디, 스릴러, 드라마)를 예측하는 트리가 있다고 가정합니다.
    *   **Target Encoding (타겟 인코딩)**: 각 영화 장르를 해당 장르를 선호하는 사용자의 '영화 시청 시간' 평균값으로 대체합니다. 예를 들어, '액션' 장르 선호자의 평균 시청 시간이 150분이라면 '액션'을 150으로 인코딩합니다. 이 방법은 정보 손실이 적고 모델이 복잡한 패턴을 학습하는 데 도움이 되지만, 학습 데이터에 과적합될 위험이 있으므로 주의해서 사용해야 합니다 (예: Cross-validation을 통한 인코딩).
    *   **Learned subset splits**: 트리 알고리즘 자체가 범주형 피처의 가능한 모든 부분 집합 분할(예: {액션, 스릴러} vs {코미디, 드라마})을 평가하여 최적의 분할을 찾아냅니다. 이는 특히 이진 분할만 가능한 트리에서 범주형 데이터를 유연하게 다루는 방법입니다.

**핵심 개념**

*   **좋은 결정 트리의 특성**:
    *   **높은 예측 성능**: 분류(classification) 문제에서는 훈련 데이터와 검증(validation) 데이터 모두에서 높은 정확도(accuracy)를 보여야 합니다. 회귀(regression) 문제에서는 낮은 RMSE(Root Mean Squared Error)를 달성해야 합니다. 이는 모델이 학습 데이터뿐만 아니라 새로운 데이터에 대해서도 잘 일반화(generalize)되어야 함을 의미합니다.
    *   **간결성 (Occam's Razor)**: 트리의 크기가 작아야(shallow) 모델을 이해하고 해석하기 쉽습니다. 복잡한 트리는 과적합(overfitting)의 위험이 있고 해석이 어렵습니다.
    *   **견고성 (Robustness)**: 입력 데이터에 작은 변화(perturbations)가 있더라도 예측 결과가 크게 변하지 않고 안정적이어야 합니다.
    *   **합리적인 분할 (Sensible Splits)**: 트리의 각 노드에서 데이터를 분할하는 기준이 도메인 지식(domain knowledge)과 일치하며 논리적으로 타당해야 합니다.
*   **편향-분산 트레이드오프 (Bias-Variance Trade-off)**:
    *   결정 트리는 모델의 깊이에 따라 편향(bias)과 분산(variance) 사이의 트레이드오프 관계를 가집니다.
    *   **깊은 트리**: 훈련 데이터의 노이즈까지 학습하여 과적합될 가능성이 높으며, 이는 높은 분산(variance)으로 이어집니다. 즉, 훈련 데이터에는 매우 잘 맞지만 새로운 데이터에는 성능이 떨어집니다.
    *   과적합을 방지하고 일반화 성능을 높이기 위해 **가지치기(pruning)**나 **앙상블(ensembles) 기법** (예: Random Forest, Gradient Boosted Decision Tree)을 사용합니다.

**코드/수식 해설**

*   **RMSE (Root Mean Squared Error)**
    회귀 문제에서 모델의 예측 오차를 평가하는 데 사용되는 지표입니다. 예측값과 실제값의 차이를 제곱하여 평균을 낸 후, 다시 제곱근을 취한 값입니다. 값이 작을수록 모델의 성능이 좋습니다.
    $$ RMSE = \sqrt{\frac{1}{N}\sum_{i=1}^{N}(y_i - \hat{y}_i)^2} $$
    여기서 $N$은 데이터 포인트의 수, $y_i$는 실제값, $\hat{y}_i$는 모델의 예측값입니다.

*   **Accuracy (정확도)**
    분류 문제에서 모델의 성능을 평가하는 데 사용되는 지표입니다. 전체 예측 중 올바르게 분류된 비율을 나타냅니다.
    $$ Accuracy = \frac{\text{Correct Predictions}}{\text{Total Predictions}} $$

*   **가지치기 (Pruning)**
    생성된 결정 트리의 복잡도를 줄이기 위해 하위 노드를 제거하거나 병합하는 기법입니다. 이를 통해 과적합을 방지하고 트리의 일반화 성능을 향상시킵니다.

*   **앙상블 (Ensembles)**
    여러 개의 약한 학습기(weak learners, 여기서는 여러 결정 트리)를 결합하여 더 강력한 예측 모델을 만드는 기법입니다.
    *   **Random Forest (RF)**: 여러 개의 독립적인 결정 트리를 생성하고, 각 트리의 예측을 집계(분류는 다수결, 회귀는 평균)하여 최종 예측을 수행합니다. 각 트리는 부트스트랩 샘플링과 특성 무작위 선택을 통해 다양성을 확보합니다.
    *   **Gradient Boosted Decision Tree (GBDT)**: 여러 개의 결정 트리를 순차적으로 학습시키며, 각 트리는 이전 트리의 오차를 보정하는 방식으로 학습됩니다. 점진적으로 모델의 성능을 향상시키는 것이 목표입니다.

**구체적 예시**

*   **높은 예측 성능 예시**: 어떤 회사가 고객의 신용 등급을 예측하는 결정 트리를 만들었다고 가정해 봅시다. 이 트리가 훈련 데이터(기존 고객 정보)에서는 99%의 정확도를 보이지만, 새로운 고객 데이터(검증 데이터)에서는 60%의 정확도밖에 내지 못한다면, 이 트리는 "좋은 트리"가 아닙니다. 훈련 데이터에 과적합되어 새로운 상황에 대한 예측 능력이 부족한 것입니다. 좋은 트리는 훈련과 검증 데이터 모두에서 안정적으로 높은 정확도를 보여야 합니다.
*   **간결성 (Occam) 예시**: 의료 진단 시스템을 위한 결정 트리를 개발한다고 할 때, 100개가 넘는 복잡한 조건으로 구성된 트리는 의사들이 결과를 신뢰하고 환자에게 설명하기 어렵습니다. 반면, "체온이 38도 이상인가?", "백혈구 수치가 특정 범위 밖인가?" 등 몇 가지 핵심 질문으로 구성된 간결한 트리는 의사들이 진단 과정을 이해하고 환자에게 납득시키기 훨씬 용이합니다.
*   **견고성 (Robustness) 예시**: 온라인 쇼핑몰에서 특정 상품의 추천 여부를 결정하는 트리가 있다고 합시다. 고객의 나이가 30세일 때 상품 A를 추천했는데, 나이를 31세로 단 1살만 바꾸니 갑자기 상품 B를 추천한다면 이 트리는 견고하지 않습니다. 작은 데이터 변화에 민감하게 반응하여 예측이 불안정하다는 의미입니다.

**시험 포인트**

*   ⭐ **좋은 결정 트리의 4가지 주요 특성**을 정확히 이해하고 설명할 수 있어야 합니다. (높은 예측 성능, 간결성, 견고성, 합리적인 분할)
*   ⭐ **편향-분산 트레이드오프 개념**과 이것이 결정 트리의 깊이(complexity)와 어떻게 연결되는지 명확히 설명할 수 있어야 합니다. (깊은 트리가 과적합, 높은 분산으로 이어짐)
*   ⭐ **과적합을 방지하고 일반화 성능을 향상시키기 위한 주요 전략**: **가지치기(pruning)**와 **앙상블 기법(Random Forest, GBDT)**의 역할과 개념을 설명할 수 있어야 합니다.

## Decision Tree Miscellany (의사결정 트리 기타 고려사항)

### 핵심 개념

이 슬라이드는 의사결정 트리를 사용할 때 고려해야 할 다양한 실용적 측면들과 한계점들을 다룹니다. 데이터 불균형 처리, 연속형 타겟 변수에서의 견고성, 과적합 방지를 위한 방법, 모델 해석 가능성, 그리고 고유한 한계점 등이 포함됩니다.

### 1. Imbalance (클래스 불균형)

*   **핵심 개념**: 분류 문제에서 특정 클래스의 데이터 수가 다른 클래스에 비해 현저히 적을 때 발생하는 문제입니다. 의사결정 트리는 이러한 불균형 데이터에 대해 소수 클래스를 제대로 학습하지 못하고 다수 클래스에 편향될 수 있습니다. 이를 해결하기 위해 불순도(impurity) 계산 시 클래스별로 가중치를 다르게 부여하는 **클래스 가중치 불순도(class-weighted impurities)** 방식이나, 데이터의 클래스 비율을 인위적으로 조정하는 **균형 샘플링(balanced sampling)** 기법을 사용할 수 있습니다.
*   **구체적 예시**: 95%가 정상인이고 5%만 사기 거래인 금융 사기 탐지 모델을 학습한다고 가정해 봅시다. 만약 모델이 모든 거래를 정상으로 예측해도 95%의 정확도를 달성하지만, 실제 사기 거래를 전혀 탐지하지 못하게 됩니다. 이때 소수 클래스(사기)에 더 높은 가중치를 부여하여 모델이 사기 거래를 더 중요하게 인식하도록 학습하거나, 사기 거래 데이터를 오버샘플링(oversampling)하여 학습 데이터의 균형을 맞출 수 있습니다.
*   **시험 포인트**: ⭐ 클래스 불균형 데이터가 의사결정 트리에 미치는 영향과 이를 완화하기 위한 두 가지 주요 방법(클래스 가중치, 균형 샘플링)을 설명하시오.

### 2. Continuous targets with heteroskedasticity (이분산성을 가진 연속형 타겟)

*   **핵심 개념**: 회귀 문제에서 타겟 변수가 연속형이고, 예측 오차의 분산(산포도)이 타겟 변수의 값에 따라 일정하지 않은 현상을 **이분산성(heteroskedasticity)**이라고 합니다. 일반적인 의사결정 회귀 트리는 노드 분할 기준으로 MSE(Mean Squared Error)를 사용하고 노드 내 평균값으로 예측합니다. 하지만 이분산성이 존재하고 이상치(outliers)가 있을 경우, MSE는 제곱 오차에 의해 이상치에 매우 민감하게 반응하여 모델의 견고성(robustness)을 떨어뜨릴 수 있습니다. 이 경우, **MAE(Mean Absolute Error)**를 분할 기준으로 사용하고, 노드 내 **중간값(median)**을 예측값으로 사용하는 것이 이상치에 덜 민감하여 더 견고한 모델을 만들 수 있습니다.
*   **코드/수식 해설**:
    *   **MSE (Mean Squared Error)**:
        $$ \text{MSE} = \frac{1}{n}\sum_{i=1}^n (y_i - \hat{y}_i)^2 $$
    *   **MAE (Mean Absolute Error)**:
        $$ \text{MAE} = \frac{1}{n}\sum_{i=1}^n |y_i - \hat{y}_i| $$
    MAE는 오차의 절대값을 사용하므로 오차가 커질수록 제곱 오차보다 증가율이 낮아 이상치에 덜 민감합니다. 노드 예측값 결정 시 MSE는 해당 노드의 타겟 변수 평균(mean)을 사용하지만, MAE는 중간값(median)을 사용합니다.
*   **구체적 예시**: 주택 가격 예측 문제에서, 고가 주택의 예측 오차 범위가 저가 주택의 예측 오차 범위보다 훨씬 넓은 경향(이분산성)을 보인다고 해봅시다. 이때 MSE를 사용하면 몇몇 아주 비싼 주택에서 발생한 큰 오차(이상치)에 모델이 과도하게 집중하여 전체 예측 성능이 저해될 수 있습니다. MAE나 중간값을 사용하면 이러한 이상치의 영향을 덜 받아 더 안정적인 예측을 수행할 수 있습니다.
*   **시험 포인트**: ⭐ 회귀 문제에서 이분산성(heteroskedasticity)이 의사결정 트리에 미치는 영향을 설명하고, MSE 대신 MAE 및 중간값을 사용하는 이유를 견고성(robustness) 측면에서 설명하시오.

### 3. Stopping vs Pruning (성장 중단 vs 가지치기)

*   **핵심 개념**: 의사결정 트리는 과도하게 성장하면 학습 데이터에 과적합(overfitting)되어 일반화 성능이 저하될 수 있습니다. 이를 방지하기 위한 두 가지 주요 전략이 있습니다.
    *   **Early Stopping (사전 가지치기/성장 중단)**: 트리를 성장시키는 도중에 미리 정의된 기준(예: 최대 깊이, 리프 노드의 최소 샘플 수, 불순도 감소의 최소 임계값 등)에 도달하면 더 이상 분할을 멈추는 방법입니다. 구현이 간단합니다.
    *   **Post-Pruning (사후 가지치기)**: 일단 트리를 최대한 깊게 성장시킨 후, 검증 데이터셋에서 성능 향상에 기여하지 않거나 오히려 성능을 저해하는 가지(서브트리)를 잘라내는 방법입니다. 일반적으로 Early Stopping보다 더 나은 일반화 성능(better generalization)을 제공하는 경우가 많지만, 계산 비용이 더 들 수 있습니다.
*   **구체적 예시**:
    *   Early Stopping: "나무를 키울 때, 너무 높이 올라가지 않도록 미리 최대 높이 제한을 두는 것."
    *   Post-Pruning: "나무를 일단 마음껏 키운 다음, 너무 무성하거나 불필요한 가지들을 나중에 솎아내어 모양을 다듬는 것."
    Post-Pruning은 트리가 성장하는 과정에서 발생할 수 있는 지역 최적해(local optimum) 문제를 피하고 더 전역적인 관점에서 최적의 구조를 찾을 기회를 제공합니다.
*   **시험 포인트**: ⭐ 의사결정 트리의 과적합을 방지하기 위한 Early Stopping과 Post-Pruning의 개념과 차이점을 설명하고, 왜 Post-Pruning이 더 좋은 일반화 성능을 제공하는 경우가 많은지 설명하시오.

### 4. Interpretability (해석 가능성)

*   **핵심 개념**: 의사결정 트리의 가장 큰 장점 중 하나는 **뛰어난 해석 가능성(Interpretability)**입니다. 루트 노드에서부터 특정 리프 노드까지의 경로(path)는 일련의 "IF-THEN" 규칙으로 명확하게 표현될 수 있으며, 이는 사람이 직관적으로 이해하기 매우 쉽습니다.
*   **구체적 예시**: 대출 승인 여부를 예측하는 의사결정 트리가 다음과 같은 규칙을 도출했다고 가정해 봅시다:
    *   "IF (나이 <= 30) AND (월 수입 > 400만원) AND (신용 등급 >= 2등급) THEN (대출 승인)"
    *   "IF (나이 > 50) AND (자산 보유액 < 1억원) THEN (대출 거절)"
    이러한 규칙은 비전문가도 모델이 어떤 기준으로 의사결정을 내렸는지 쉽게 파악할 수 있도록 해줍니다.
*   **시험 포인트**: ⭐ 의사결정 트리의 주요 장점인 해석 가능성(Interpretability)을 설명하고, "IF-THEN" 규칙을 사용하여 그 특징을 구체적으로 기술하시오.

### 5. Limitations (한계점)

*   **핵심 개념**:
    *   **Axis-aligned splits (축 정렬 분할)**: 대부분의 표준 의사결정 트리는 각 노드에서 특정 단일 피처의 임계값에 따라 데이터를 분할합니다. 이는 분할 경계가 좌표축에 평행하게 만들어진다는 것을 의미합니다. 이러한 방식은 데이터 공간에서 대각선이나 복잡한 비선형 결정 경계를 효율적으로 표현하기 어렵게 만듭니다. 여러 개의 축 정렬 분할을 계단식으로 조합해야 하므로 트리가 불필요하게 깊고 복잡해질 수 있습니다.
    *   **Oblique trees (경사 트리)**: 축 정렬 분할의 한계를 극복하기 위해 제안된 것이 경사 트리입니다. 경사 트리는 노드 분할 시 단일 피처의 임계값이 아닌, 여러 피처의 선형 조합(linear predicate)을 사용합니다 (예: $w_1 X_1 + w_2 X_2 < \text{threshold}$). 이는 좌표축에 평행하지 않은 **비축 정렬(non-axis-aligned)** 분할을 가능하게 하여 더 복잡하고 효율적인 결정 경계를 모델링할 수 있습니다. 그러나 최적의 선형 조합(가중치 $w_i$)을 찾는 과정이 훨씬 복잡하고 계산 비용이 많이 들어, 일반적인 축 정렬 트리에 비해 훈련(train)하기 어렵습니다.
*   **코드/수식 해설**:
    *   **축 정렬 분할 예시**: $X_1 \le \text{threshold}_1$
    *   **경사 분할 예시**: $\sum_{i=1}^D w_i X_i \le \text{threshold}$ (여기서 $w_i$는 가중치)
*   **구체적 예시**: 2차원 평면에서 데이터 포인트들이 대각선을 기준으로 두 클래스로 명확하게 나뉘어 있는 상황을 생각해 봅시다. 축 정렬 트리는 이 대각선 경계를 모방하기 위해 여러 개의 가로/세로 분할을 계단식으로 만들어야 하므로 트리가 비효율적이고 복잡해집니다. 반면 경사 트리는 단 한 번의 대각선 분할로 데이터를 효율적으로 나눌 수 있습니다.
*   **시험 포인트**: ⭐ 의사결정 트리의 '축 정렬 분할(axis-aligned splits)' 방식의 한계를 설명하고, 이를 극복하기 위한 '경사 트리(oblique trees)'의 개념과 그 장단점을 비교하여 설명하시오.

### Decision Trees vs k-NN (k-Nearest Neighbors) 비교

이 슬라이드는 두 가지 주요 분류 모델인 Decision Tree(결정 트리)와 k-NN(k-최근접 이웃)을 여러 측면에서 비교하여 각 모델의 특징과 장단점을 설명합니다.

---

### **핵심 개념**

*   **모델 (Model)**:
    *   **Decision Tree**: 데이터를 학습하여 전체 특징 공간을 명시적인 규칙 기반으로 분할하는 **전역적 모델(Global partition)**입니다. 이 모델은 학습 단계를 거쳐 '규칙'을 만듭니다.
    *   **k-NN**: 훈련 데이터를 모두 저장하고 새로운 데이터 포인트가 들어왔을 때 가장 가까운 $k$개의 이웃을 찾아 분류하는 **인스턴스 기반(Instance-based)** 모델입니다. 별도의 '학습' 단계가 없으며, 훈련 데이터 자체가 모델 역할을 합니다.
    *   ⭐ **시험 포인트**: Decision Tree는 학습된 규칙(explicit model)을 가지지만, k-NN은 훈련 데이터를 그대로 사용하는 비모수적(non-parametric) 인스턴스 기반 모델입니다.

*   **결정 경계 (Decision boundary)**:
    *   **Decision Tree**: 각 특징 축에 평행하게 데이터를 분할하여 **축에 정렬된 직사각형(Axis-aligned rectangles)** 형태의 결정 경계를 만듭니다. 이는 각 영역에서 예측값이 일정한 **조각별 상수(piecewise constant)** 형태를 가집니다.
    *   **k-NN**: 훈련 데이터 포인트에 따라 결정 경계가 암묵적으로 정의되며, 매우 유연하고 복잡한 **보로노이 영역(Voronoi regions)** 형태를 가집니다. 이 경계는 모든 점이 해당 클래스의 최근접 이웃이 되는 영역을 나타냅니다.
    *   ⭐ **시험 포인트**: Decision Tree의 결정 경계는 직관적인 직사각형 형태이며 비교적 단순하지만, k-NN은 데이터 분포에 따라 매우 유연하고 복잡한 형태를 띨 수 있습니다.

*   **학습 시간 (Train time)**:
    *   **Decision Tree**: 최적의 분할 지점을 찾기 위해 특징 값들을 정렬하고 스캔하는 과정이 필요하므로 **보통(Moderate)** 수준의 시간이 소요됩니다.
    *   **k-NN**: 훈련 데이터를 단순히 저장하는 것이 전부이므로 학습 시간은 **미미(Trivial)**합니다. (실질적인 학습 단계가 없다고 볼 수 있습니다.)

*   **예측 시간 (Test time)**:
    *   **Decision Tree**: 트리의 뿌리(root)에서 잎(leaf)까지 경로를 따라가기 때문에 트리의 깊이에 비례하는 시간 복잡도($O(\text{depth})$)를 가집니다.
    *   **k-NN**: 새로운 인스턴스에 대해 모든 $N$개의 훈련 데이터와의 거리를 계산해야 하므로, $D$차원의 데이터인 경우 $O(ND)$의 시간 복잡도를 가집니다. 특정 인덱싱 기법(예: k-d tree, Ball tree)을 사용하지 않는 한 이 비용은 높을 수 있습니다.
    *   ⭐ **시험 포인트**: k-NN은 학습 시간은 짧지만 예측 시간이 데이터의 크기($N$)와 차원($D$)에 따라 크게 증가할 수 있다는 단점이 있습니다. Decision Tree는 예측 시간이 트리의 깊이에 따라 빠릅니다.

*   **피처 처리 (Features)**:
    *   **Decision Tree**: 수치형(numerical)과 범주형(categorical) 등 **혼합된 피처 타입을 별도의 전처리 없이 기본적으로 처리**할 수 있습니다.
    *   **k-NN**: 거리 계산이 핵심이므로, 모든 피처가 **거리 계산이 가능한 수치형 형식**이어야 합니다. 따라서 범주형 피처는 원-핫 인코딩(one-hot encoding)과 같은 전처리 과정이 필요합니다.

*   **해석 가능성 (Interpretability)**:
    *   **Decision Tree**: 트리의 각 노드와 가지가 특정 조건에 따른 규칙을 나타내므로, 최종 결정 경로를 **if-then 규칙 형태로 쉽게 이해하고 설명할 수 있어 해석 가능성이 높습니다.**
    *   **k-NN**: 특정 예측이 가장 가까운 훈련 인스턴스에 기반하므로, 일반적인 규칙을 추출하기 어렵고 **해석 가능성이 낮습니다.**

*   **민감도 (Sensitivity)**:
    *   **Decision Tree**: 가지치기(pruning) 과정을 통해 과적합을 방지하고 **국소적인 노이즈에 대해 강건(robust)**할 수 있습니다.
    *   **k-NN**: 거리 기반 모델이므로 **피처 스케일링(scaling)이나 이상치(noise)에 매우 민감**합니다. 피처 스케일이 적절하지 않으면 특정 피처가 거리에 과도하게 영향을 미칠 수 있습니다.
    *   ⭐ **시험 포인트**: k-NN 사용 시 피처 스케일링은 필수적입니다.

### **코드/수식 해설**

*   **k-NN 예측 시간 복잡도**:
    새로운 데이터 $x_{new}$가 주어졌을 때, 훈련 데이터셋 $D_{train} = \{(x_i, y_i)\}_{i=1}^N$에 대해 다음과 같은 단계를 거칩니다.
    1.  모든 훈련 데이터 $x_i$에 대해 $x_{new}$와의 거리 $d(x_{new}, x_i)$를 계산합니다. 이는 $N$번의 거리 계산이 필요하며, 각 거리 계산은 $D$차원 벡터에 대한 연산이므로 $O(D)$의 시간이 소요됩니다.
    2.  따라서 총 거리 계산 시간은 $O(ND)$가 됩니다.
    3.  이후 가장 가까운 $k$개의 이웃을 찾는 과정은 정렬 또는 우선순위 큐를 통해 효율적으로 처리될 수 있습니다.

    $$ \text{Test Time (k-NN, unindexed)} = O(ND) $$
    여기서 $N$은 훈련 데이터의 수, $D$는 피처의 차원입니다.

*   **Decision Tree 예측 시간 복잡도**:
    Decision Tree의 예측 과정은 트리의 루트 노드에서 시작하여 리프 노드에 도달할 때까지 분할 조건을 따라 내려가는 과정입니다. 이 과정은 트리의 깊이에 비례합니다.
    $$ \text{Test Time (Decision Tree)} = O(\text{depth}) $$
    여기서 $\text{depth}$는 트리의 최대 깊이입니다. 일반적인 균형 이진 트리(balanced binary tree)의 경우 $\text{depth} \approx \log_2 N$이 될 수 있습니다.

### **구체적 예시**

*   **Decision Tree 예시 (직사각형 결정 경계)**:
    "고객의 나이와 월 소득에 따라 대출 승인 여부를 결정하는 모델"을 Decision Tree로 만든다면, "나이 < 30세" 또는 "월 소득 > 500만원"과 같은 규칙으로 데이터를 분할합니다. 이를 시각화하면, 나이 축과 소득 축에 평행한 선들로 나누어진 직사각형 영역들이 각각 '대출 승인' 또는 '대출 거절'로 분류됩니다.

*   **k-NN 예시 (보로노이 영역)**:
    "새로운 품종의 꽃이 주어졌을 때, 기존 꽃들의 꽃잎 길이와 너비를 바탕으로 품종을 분류"하는 경우를 생각해봅시다. 새로운 꽃이 주어지면, 그래프 상에서 가장 가까운 $k$개의 훈련 꽃들을 찾습니다. 만약 $k=3$이고, 가까운 3개 중 2개가 'A품종'이고 1개가 'B품종'이라면, 새로운 꽃은 'A품종'으로 분류됩니다. 이때 각 훈련 데이터 포인트 주변의 '영향권'이 보로노이 영역을 형성하며, 이 영향권의 경계들이 암묵적인 결정 경계가 됩니다.

*   **피처 전처리 예시**:
    "성별(범주형: 남/여), 나이(수치형), 수입(수치형)" 데이터를 사용하여 분류한다고 가정합니다.
    *   **Decision Tree**: 성별을 그대로 사용하여 '성별 = 남'과 같이 분할할 수 있습니다.
    *   **k-NN**: 성별을 '남=0, 여=1' 또는 원-핫 인코딩(남=\[1,0], 여=\[0,1])으로 변환해야 거리 계산이 가능해집니다. 또한 나이와 수입 피처는 스케일이 크게 다를 수 있으므로, 표준화(Standardization)나 정규화(Normalization)와 같은 스케일링 작업이 필수적입니다. 예를 들어, 나이(20-60)와 수입(100만원-1000만원)을 스케일링하지 않으면 수입 피처가 거리 계산에 지배적인 영향을 미칠 것입니다.

### **시험 포인트**

*   ⭐ **Decision Tree는 명시적인 규칙 기반 모델, k-NN은 인스턴스 기반 비모수 모델이다.**
*   ⭐ **Decision Tree의 결정 경계는 축에 정렬된 직사각형 형태, k-NN은 유연한 보로노이 영역이다.**
*   ⭐ **k-NN은 학습 시간이 매우 짧지만, 예측 시간이 데이터의 양($N$)과 차원($D$)에 따라 $O(ND)$로 증가할 수 있다.**
*   ⭐ **Decision Tree는 혼합된 피처 타입을 자연스럽게 처리하지만, k-NN은 거리 계산을 위해 피처 전처리(인코딩, 스케일링)가 필수적이다.**
*   ⭐ **Decision Tree는 규칙 기반으로 해석력이 높은 반면, k-NN은 인스턴스 기반으로 해석력이 낮다.**
*   ⭐ **k-NN은 피처 스케일링과 노이즈에 매우 민감하므로 전처리가 중요하다. Decision Tree는 가지치기를 통해 노이즈에 강건할 수 있다.**

**핵심 개념**

*   **데이터로부터 엔트로피 추정 (Estimating Entropy from Data)**: 결정 트리(Decision Tree)와 같은 머신러닝 모델에서 특정 노드의 불확실성 또는 불순도(impurity)를 측정하기 위해 엔트로피를 사용합니다. 실제 확률 분포를 모를 때, 주어진 데이터 샘플의 클래스 비율을 이용하여 엔트로피를 추정하는 방법을 학습합니다. 이는 해당 노드가 얼마나 "혼합되어" 있는지, 즉 여러 클래스의 샘플이 얼마나 균등하게 분포되어 있는지를 나타냅니다.
*   **플러그-인 추정량 (Plug-in Estimate)**: 실제 확률 분포 $P(X)$를 모를 때, 관측된 데이터를 기반으로 확률 분포를 $\hat{P}(X)$로 추정한 후, 이 추정치를 원래 공식에 대입하여 원하는 통계량(여기서는 엔트로피)을 계산하는 방법입니다.
*   **분할에 대한 불순도 감소 (Impurity Drop for Split)**: 결정 트리가 노드를 분할할 때, 어떤 분할이 가장 좋은지를 평가하는 지표입니다. 부모 노드의 불순도에서 자식 노드들의 불순도를 가중 평균하여 뺀 값으로, 정보 이득(Information Gain)이라고도 불립니다. 불순도 감소 값이 클수록 해당 분할이 노드의 순수도(purity)를 크게 향상시키는 좋은 분할임을 의미합니다.

**코드/수식 해설**

1.  **플러그-인 엔트로피 추정량 ($\hat{H}$)**:
    데이터가 주어진 노드에서 클래스 $k$에 속하는 샘플의 개수가 $n_k$이고, 해당 노드의 총 샘플 개수가 $n = \sum_k n_k$일 때, 클래스 $k$의 확률 추정치는 다음과 같습니다.

    $$ \hat{p}_k = \frac{n_k}{n} $$

    이를 이용하여 엔트로피를 추정합니다.

    $$ \hat{H} = -\sum_k \hat{p}_k \log_2 \hat{p}_k $$

    *   $\hat{p}_k$: 클래스 $k$가 나타날 확률의 추정치.
    *   $\log_2$: 밑이 2인 로그를 사용하며, 정보량을 비트(bit) 단위로 측정함을 의미합니다.
    *   $\sum_k$: 모든 클래스 $k$에 대한 합을 의미합니다.
    *   $\hat{H}$: 추정된 엔트로피 값으로, 노드의 불순도를 나타냅니다. 엔트로피가 0이면 완벽하게 순수한 노드(모든 샘플이 한 클래스에 속함)이고, 엔트로피가 높으면 불순도가 높습니다.

2.  **분할 $s$에 대한 불순도 감소 ($\Delta H(s)$)**:
    특정 분할 $s$가 부모 노드를 왼쪽 자식 노드와 오른쪽 자식 노드로 나눌 때, 이 분할로 인해 얻는 불순도 감소량은 다음과 같이 계산됩니다.

    $$ \Delta H(s) = \hat{H}_{\text{parent}} - \frac{n_L}{n} \hat{H}_L - \frac{n_R}{n} \hat{H}_R $$

    *   $\hat{H}_{\text{parent}}$: 분할 전 부모 노드의 엔트로피.
    *   $n_L, n_R$: 각각 왼쪽 자식 노드와 오른쪽 자식 노드의 샘플 개수.
    *   $n$: 부모 노드의 총 샘플 개수 ($n = n_L + n_R$).
    *   $\hat{H}_L, \hat{H}_R$: 각각 왼쪽 자식 노드와 오른쪽 자식 노드의 엔트로피.
    *   $\frac{n_L}{n}, \frac{n_R}{n}$: 각 자식 노드가 부모 노드에서 차지하는 비율로, 자식 노드의 엔트로피를 가중 평균하는 데 사용됩니다.
    *   $\Delta H(s)$: 분할 $s$에 의해 감소하는 불순도(정보 이득). 이 값이 클수록 해당 분할이 노드를 더 순수하게 만듭니다.

3.  **지니 불순도(Gini) 및 MSE(Mean Squared Error)**:
    위의 엔트로피와 정보 이득 계산 방식은 다른 불순도 지표에도 유사하게 적용됩니다.
    *   **지니 불순도**: 분류(classification) 문제에서 엔트로피와 함께 널리 사용되는 불순도 지표입니다.
    *   **평균 제곱 오차 (MSE)**: 회귀(regression) 문제에서 노드의 불순도를 측정하는 데 사용되며, 예측값과 실제값 차이의 제곱 평균을 나타냅니다.

**구체적 예시**

어떤 노드에 총 100개의 데이터 샘플이 있고, 이 중 70개는 클래스 '고양이', 30개는 클래스 '개'라고 가정해 봅시다.

1.  **부모 노드의 엔트로피 계산**:
    *   $n_{\text{고양이}} = 70, n_{\text{개}} = 30, n = 100$.
    *   $\hat{p}_{\text{고양이}} = 70/100 = 0.7$
    *   $\hat{p}_{\text{개}} = 30/100 = 0.3$
    *   $\hat{H}_{\text{parent}} = -(0.7 \log_2 0.7 + 0.3 \log_2 0.3)$
        *   $\log_2 0.7 \approx -0.5146$, $\log_2 0.3 \approx -1.7370$
        *   $\hat{H}_{\text{parent}} = -(0.7 \times -0.5146 + 0.3 \times -1.7370) = -(-0.36022 - 0.5211) \approx 0.8813$ 비트

2.  **분할 후 불순도 감소 계산 (가상 시나리오)**:
    이제 이 노드를 "털 색깔이 갈색인가?"라는 기준으로 분할했다고 가정합시다.
    *   **왼쪽 자식 노드 (털 색깔이 갈색이 아님)**: 60개 샘플 중 '고양이' 60개, '개' 0개.
        *   $n_L = 60$. $\hat{H}_L = -(1.0 \log_2 1.0 + 0 \log_2 0) = 0$ (완전히 순수)
    *   **오른쪽 자식 노드 (털 색깔이 갈색임)**: 40개 샘플 중 '고양이' 10개, '개' 30개.
        *   $n_R = 40$. $\hat{p}_{\text{고양이}} = 10/40 = 0.25$, $\hat{p}_{\text{개}} = 30/40 = 0.75$
        *   $\hat{H}_R = -(0.25 \log_2 0.25 + 0.75 \log_2 0.75)$
            *   $\log_2 0.25 = -2$, $\log_2 0.75 \approx -0.4150$
            *   $\hat{H}_R = -(0.25 \times -2 + 0.75 \times -0.4150) = -(-0.5 - 0.31125) \approx 0.8113$ 비트
    *   **불순도 감소 계산**:
        $\Delta H(s) = \hat{H}_{\text{parent}} - \frac{n_L}{n} \hat{H}_L - \frac{n_R}{n} \hat{H}_R$
        $\Delta H(s) = 0.8813 - \frac{60}{100} \times 0 - \frac{40}{100} \times 0.8113$
        $\Delta H(s) = 0.8813 - 0 - 0.32452 \approx 0.5568$

이 분할은 약 0.5568 비트의 불순도를 감소시켰으며, 이는 상당히 유용한 분할임을 나타냅니다. 결정 트리는 이렇게 계산된 불순도 감소량이 최대가 되는 분할 기준을 선택하여 트리를 성장시킵니다.

**시험 포인트**

*   ⭐ **엔트로피의 정의 및 계산 방법**: $\hat{H} = -\sum_k \hat{p}_k \log_2 \hat{p}_k$ 공식을 정확히 이해하고, 주어진 데이터(각 클래스 샘플 수)로부터 $\hat{p}_k$를 계산하여 엔트로피 값을 구할 수 있어야 합니다. 특히 $\hat{p}_k = n_k/n$ 임을 명확히 알아야 합니다.
*   ⭐ **불순도 감소 (정보 이득)의 정의 및 계산 방법**: $\Delta H(s) = \hat{H}_{\text{parent}} - \frac{n_L}{n} \hat{H}_L - \frac{n_R}{n} \hat{H}_R$ 공식을 이해하고, 부모 및 자식 노드의 엔트로피와 샘플 수를 이용하여 정보 이득을 계산할 수 있어야 합니다. 가중 평균의 개념이 중요합니다.
*   ⭐ **결정 트리에서 엔트로피/정보 이득이 사용되는 목적**: 노드의 불확실성(불순도)을 측정하고, 최적의 분할 기준(가장 큰 정보 이득을 주는 분할)을 찾기 위해 사용된다는 점을 설명할 수 있어야 합니다.
*   로그의 밑이 2인 이유와 엔트로피 값이 0일 때의 의미 (완벽한 순수도)를 알아두세요.
*   엔트로피 외에 지니 불순도, MSE와 같은 다른 불순도 지표들도 있다는 점을 기억해야 합니다 (특히 분류: 지니 불순도, 회귀: MSE).

**핵심 개념**:
*   **Cost-Complexity Pruning (비용-복잡도 가지치기)**: 의사결정 나무(Decision Tree) 모델이 훈련 데이터에 과적합(overfitting)되는 것을 방지하기 위한 가지치기(pruning) 방법 중 하나입니다. 트리의 복잡도(주로 리프 노드 수)와 훈련 데이터에 대한 오차(경험적 위험)를 동시에 고려하여, 모델의 일반화 성능을 최대화하는 최적의 트리를 찾는 기법입니다.

**코드/수식 해설**:
*   **부분 트리 비용 정의**:
    $$R_\alpha(T) = R(T) + \alpha |T|$$
    *   $R_\alpha(T)$: 특정 복잡도 파라미터 $\alpha$에 대한 부분 트리 $T$의 총 비용(Cost)을 나타냅니다. 이 값을 최소화하는 트리를 찾는 것이 목표입니다.
    *   $R(T)$: 트리가 훈련 데이터에 대해 가지는 **경험적 위험(empirical risk)**입니다. 분류 문제에서는 주로 **오분류율(misclassification rate)**을 사용합니다. 트리가 더 복잡할수록 (가지치기가 덜 될수록) 훈련 데이터에 대한 경험적 위험은 일반적으로 감소합니다.
    *   $\alpha$: **복잡도 파라미터(complexity parameter)** 또는 **벌칙 계수(penalty parameter)**입니다. $\alpha \ge 0$인 스칼라 값으로, 트리의 복잡도 $|T|$에 대한 중요도를 조절합니다. $\alpha$ 값이 커질수록 트리의 복잡도에 더 큰 벌칙을 부여하게 됩니다.
    *   $|T|$: 부분 트리 $T$의 **리프 노드(leaf node) 수**를 의미하며, 트리의 복잡도를 나타내는 척도입니다. 리프 노드 수가 많을수록 트리는 더 깊고 복잡합니다.

*   **동작 원리**:
    *   위 수식은 트리의 **경험적 위험**과 **복잡도** 사이의 균형을 맞추는 목적 함수(objective function)입니다.
    *   $\alpha$ 값이 0일 때는 트리의 복잡도에 대한 벌칙이 없으므로, 최대한 복잡하고 훈련 데이터에 완벽히 맞는 트리를 생성하려고 합니다 (과적합 위험).
    *   $\alpha$ 값이 증가할수록 $|T|$에 대한 벌칙이 커지므로, 시스템은 $R_\alpha(T)$를 최소화하기 위해 리프 노드 수가 적은(즉, 더 많이 가지치기된) 트리를 선호하게 됩니다.
    *   $\alpha$를 0부터 점진적으로 증가시키면서 최적의 부분 트리를 찾으면, 이 트리들은 **중첩된 순서(nested sequence)**를 이룹니다. 즉, $\alpha$가 커질수록 이전보다 더 작고 간단한 트리를 얻게 됩니다.
    *   이렇게 다양한 $\alpha$ 값에 대해 생성된 후보 트리들 중에서 최종 최적의 트리는 **검증 세트(validation set)** 또는 **교차 검증(Cross-Validation, CV)**을 사용하여 선택합니다. 이는 모델이 훈련 데이터뿐만 아니라 새로운, 보지 못한 데이터에 대해서도 얼마나 잘 작동하는지(일반화 성능) 평가하기 위함입니다.

**구체적 예시**:
*   **실생활 비유**: 회사의 프로젝트 관리자가 프로젝트의 성공(낮은 실패율)과 자원 소모(인력, 시간) 사이에서 균형을 찾아야 한다고 가정해 봅시다.
    *   $R(T)$: 프로젝트의 실패율 (낮을수록 좋음).
    *   $|T|$: 프로젝트에 투입된 인력 수 (많을수록 복잡함).
    *   $\alpha$: '효율성'을 얼마나 중요하게 생각하는지에 대한 가중치.
    *   $R_\alpha(T)$: 프로젝트의 '총 비용 지수'.
    *   만약 $\alpha$가 낮으면 (효율성을 덜 중요시), 실패율만 낮출 수 있다면 많은 인력을 투입하는 복잡한 프로젝트를 선호할 수 있습니다.
    *   반대로 $\alpha$가 높으면 (효율성을 매우 중요시), 실패율이 조금 높아지더라도 적은 인력으로 진행되는 간단한 프로젝트를 선호하여 총 비용 지수를 낮추려고 할 것입니다.
    *   가지치기는 $\alpha$를 바꿔가며 다양한 규모의 프로젝트를 시뮬레이션하고, 실제 시장 반응(검증/CV)을 통해 가장 이상적인 프로젝트 규모를 결정하는 것과 유사합니다.

*   **Scikit-learn을 이용한 예시**: `sklearn.tree.DecisionTreeClassifier`에서는 `ccp_alpha` 파라미터를 통해 비용-복잡도 가지치기를 구현할 수 있습니다.
    ```python
    from sklearn.tree import DecisionTreeClassifier
    from sklearn.model_selection import train_test_split, cross_val_score
    from sklearn.datasets import load_iris
    import numpy as np

    # 붓꽃 데이터 로드
    iris = load_iris()
    X, y = iris.data, iris.target

    # 훈련 세트와 테스트 세트 분리
    X_train, X_test, y_train, y_test = train_test_split(X, y, random_state=42)

    # ccp_alpha가 0일 때 (가지치기 없음)
    clf_no_pruning = DecisionTreeClassifier(random_state=42, ccp_alpha=0)
    clf_no_pruning.fit(X_train, y_train)
    print(f"No pruning (ccp_alpha=0) - Test accuracy: {clf_no_pruning.score(X_test, y_test):.2f}")
    print(f"No pruning - Number of leaves: {clf_no_pruning.get_n_leaves()}")

    # ccp_alpha 값들을 탐색하여 최적의 트리 찾기
    # `cost_complexity_pruning_path`를 사용해 유효한 alpha 값들을 얻을 수 있습니다.
    path = clf_no_pruning.cost_complexity_pruning_path(X_train, y_train)
    ccp_alphas = path.ccp_alphas

    # 교차 검증을 통해 최적의 ccp_alpha 선택
    alpha_accuracies = []
    for ccp_alpha in ccp_alphas:
        clf = DecisionTreeClassifier(random_state=42, ccp_alpha=ccp_alpha)
        # 5-fold 교차 검증 수행
        scores = cross_val_score(clf, X_train, y_train, cv=5)
        alpha_accuracies.append(np.mean(scores))

    # 가장 높은 교차 검증 정확도를 보인 ccp_alpha 선택
    optimal_alpha_idx = np.argmax(alpha_accuracies)
    optimal_ccp_alpha = ccp_alphas[optimal_alpha_idx]
    print(f"\nOptimal ccp_alpha from CV: {optimal_ccp_alpha:.4f}")

    # 최적의 ccp_alpha로 트리 다시 학습 및 평가
    clf_pruned = DecisionTreeClassifier(random_state=42, ccp_alpha=optimal_ccp_alpha)
    clf_pruned.fit(X_train, y_train)
    print(f"Pruned (ccp_alpha={optimal_ccp_alpha:.4f}) - Test accuracy: {clf_pruned.score(X_test, y_test):.2f}")
    print(f"Pruned - Number of leaves: {clf_pruned.get_n_leaves()}")
    ```
    이 코드는 가지치기가 없는 트리와 비용-복잡도 가지치기를 통해 얻은 트리의 테스트 정확도와 리프 노드 수를 비교하여, 적절한 $\alpha$ 값 선택이 모델의 일반화 성능에 어떤 영향을 미치는지 보여줍니다. 일반적으로 가지치기된 트리가 더 적은 리프 노드를 가지면서도 더 좋은(또는 비슷한) 테스트 정확도를 보일 수 있습니다.

**시험 포인트**:
*   ⭐ **Cost-Complexity Pruning의 핵심 목표**: 의사결정 나무의 **과적합(overfitting)**을 방지하고, **일반화 성능(generalization performance)**을 향상시키는 것입니다.
*   ⭐ **비용 함수 $R_\alpha(T) = R(T) + \alpha |T|$ 의 각 항의 의미**:
    *   $R(T)$: 훈련 데이터에 대한 **경험적 위험(empirical risk)** 또는 오차 (예: 오분류율).
    *   $\alpha$: **복잡도 파라미터**로, 트리의 복잡도에 대한 **벌칙(penalty)의 크기**를 조절합니다.
    *   $|T|$: 트리의 **리프 노드(leaf node) 수**로, 트리의 복잡도를 나타내는 지표입니다.
*   ⭐ **$\alpha$ 값이 증가할 때의 효과**: $\alpha$가 커질수록 트리의 복잡도에 부여되는 벌칙이 커져, **더 많이 가지치기된(더 간단한)** 트리가 선택될 가능성이 높아집니다.
*   ⭐ **최적의 $\alpha$ 선택 방법**: 훈련 데이터가 아닌 **검증 세트(validation set)** 또는 **교차 검증(Cross-Validation, CV)**을 사용하여 모델의 일반화 성능이 가장 좋은 $\alpha$ 값을 선택합니다.
*   ⭐ **scikit-learn에서의 구현**: `DecisionTreeClassifier`의 `ccp_alpha` 파라미터를 통해 비용-복잡도 가지치기를 제어할 수 있습니다.

### 핵심 개념
이 슬라이드는 의사 결정 트리(Decision Tree)의 구성 문제에 대한 근본적인 이론적 한계와 이를 극복하기 위한 실용적인 접근 방식들을 요약하고 있습니다.

1.  **최적 의사 결정 트리 구성의 NP-완전성**:
    *   Hyifil & Rivest (1976)의 연구에 따르면, 모든 가능한 의사 결정 트리 중에서 **가장 최적의(optimal) 이진 의사 결정 트리를 구성하는 문제는 NP-완전(NP-complete)**입니다.
    *   이는 주어진 데이터에 대해 가장 오류가 적거나 특정 기준을 가장 잘 만족하는 트리를 찾는 것이, 입력 데이터의 크기가 커질수록 현실적으로 해결 불가능할 정도로 많은 계산 시간(지수 시간)을 요구한다는 의미입니다. 즉, 브루트 포스(brute force)로 모든 경우의 수를 탐색하는 것은 사실상 불가능합니다.

2.  **휴리스틱 기반의 트리 구성 알고리즘**:
    *   최적의 트리를 찾는 것이 NP-완전 문제이기 때문에, 실제 데이터 분석에서는 **탐욕적(greedy)이고 휴리스틱(heuristic) 기반의 알고리즘**을 사용하여 의사 결정 트리를 구성합니다. 이 알고리즘들은 전역 최적해(global optimum)를 보장하지는 않지만, 실용적인 시간 내에 충분히 좋은(sub-optimal) 트리를 생성합니다.
    *   **ID3 (Iterative Dichotomiser 3)**: Ross Quinlan이 개발한 초기 알고리즘으로, 정보 이득(Information Gain)을 분할 기준으로 사용합니다. 주로 범주형 데이터를 처리합니다.
    *   **C4.5**: ID3의 후속 버전으로, Quinlan이 개발했습니다. 정보 이득률(Gain Ratio)을 사용하여 정보 이득의 편향성을 보완하고, 연속형 변수 처리, 결측값 처리, 가지치기(pruning) 기능이 추가되었습니다.
    *   **CART (Classification and Regression Trees)**: Breiman 등이 개발한 알고리즘으로, 분류(Classification)와 회귀(Regression) 모두에 사용될 수 있습니다. 분류 트리에서는 지니 불순도(Gini impurity)를, 회귀 트리에서는 평균 제곱 오차(Mean Squared Error)를 분할 기준으로 사용합니다. 연속형 변수를 효율적으로 처리하며, 이진 분할(binary split)을 특징으로 합니다.

### 구체적 예시
*   **NP-완전성의 비유**: 여러분이 복잡한 미로에서 가장 짧은 경로를 찾는다고 가정해 봅시다. 가장 짧은 경로를 *정확히* 찾아내려면 모든 가능한 경로를 하나하나 탐색하고 비교해야 합니다. 미로가 커질수록 이 작업은 기하급수적으로 어려워집니다. 이것이 NP-완전 문제와 유사합니다. 대신, 우리는 눈앞의 선택지 중에서 가장 좋아 보이는 방향(휴리스틱)으로 나아가는 방식으로 '충분히 좋은' 경로를 찾습니다.
*   **ID3, C4.5, CART의 역할**: 이 알고리즘들은 위 비유에서 '눈앞의 선택지 중 가장 좋아 보이는 방향'을 결정하는 구체적인 규칙(예: 정보 이득이 가장 높은 특성으로 노드 분할)을 제공하는 역할을 합니다. 예를 들어, CART는 각 노드에서 데이터를 가장 잘 분리하는 특성(feature)과 임계값(threshold)을 탐욕적으로 찾아서 트리를 아래로 확장해 나갑니다. 이 과정은 글로벌 최적을 보장하지 않지만, 현실적인 시간 내에 데이터 패턴을 잘 학습하는 트리를 만듭니다.

### 시험 포인트
*   ⭐ **최적 의사 결정 트리 구성이 NP-완전 문제임을 이해하는 것이 중요합니다.** 이 이론적 한계 때문에 실제 알고리즘들이 어떤 방식으로 설계되었는지(탐욕적, 휴리스틱) 연결 지어 설명할 수 있어야 합니다.
*   ⭐ **ID3, C4.5, CART가 의사 결정 트리 학습의 주요 알고리즘이며, 이들이 왜 개발되었는지(NP-완전성 극복을 위한 실용적 접근)를 알고 있어야 합니다.** 각 알고리즘의 주요 특징(예: 분할 기준, 처리 가능한 데이터 타입)을 간략하게 설명할 수 있다면 더욱 좋습니다.
*   ⭐ **NP-완전성의 의미를 다른 NP 문제들과 연관 지어 설명할 수 있어야 합니다.** (예: "외판원 문제"와 같은 다른 NP-완전 문제와의 유사성을 인지).

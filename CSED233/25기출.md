## 1번 — 포인터 · 배열 · 캐스팅 ⚠️

> **복기 메모:** 코드 전체를 복구하지 못함. 아래는 복기된 힌트와 PPT 내용 기반으로 재구성한 유사 코드.

### (a) ~ (c) 배열·포인터 출력

```cpp
int arr[] = {1, 5, 10, 20, 30};
int *p = arr[0];   // ⚠️ 실제 코드와 다를 수 있음

cout << p;         // (a) arr의 시작 주소 출력
p = arr;
cout << p;         // (b) arr의 시작 주소 출력 (동일)
cout << (p++);     // (c) 현재 p 값 출력 후 p 증가 (후위 증가)
cout << p;         // (d) 증가된 p 값 출력
cout << (++p);     // (e) p를 먼저 증가시킨 후 출력 (전위 증가)
```

**핵심 개념:**
- `p = arr` → p가 arr[0]의 주소를 가리킴 (배열 이름 = 첫 원소 주소)
- `p[i]` ≡ `*(p + i)` (배열 인덱싱 = 포인터 연산)
- `p++` : 현재 값 반환 후 포인터를 다음 원소로 이동 (int → +4 bytes)
- `++p` : 먼저 이동한 후 값 반환

### (d) ~ (f) 캐스팅

```cpp
// (d) int를 float으로 나누면 반환형?
int a = 7;
float b = 2.0f;
auto result = a / b;   // → float (3.5f)

// (e) int형 변수에 (int / float) 결과를 대입하면?
int x = (7 / 2.0f);   // → 3 (소수점 버림, truncation)

// (f) &arr[3]이 의미하는 바
int arr[] = {1, 5, 10, 20, 30};
// &arr[3] : arr의 네 번째 원소(값 20)의 메모리 주소
```

**정리:**
| 표현식 | 타입 | 값 |
|--------|------|----|
| `7 / 2.0f` | `float` | `3.5` |
| `(int)(7 / 2.0f)` | `int` | `3` |
| `&arr[3]` | `int*` | arr 시작주소 + 3×sizeof(int) |

## 2번 — T(n) 계산 & Big-Oh / Big-Omega / Big-Theta

### (1) T(n) 구하기

> **정답: T(n) = 2n − 1**

코드 구조 (복기 기반 재구성):
```cpp
// 루프 n회 × 내부 연산 2번 = 2n회, 루프 외 연산 -1 등 → 정확한 T(n) = 2n-1
```

### (2) 각 조건 판별

**형식적 정의:**

| 표기 | 조건 | 의미 |
|------|------|------|
| T(n) = O(g(n)) | ∃ c > 0, n₀ > 0 s.t. **T(n) ≤ c·g(n)** for all n ≥ n₀ | 상한 (Upper Bound) |
| T(n) = Ω(g(n)) | ∃ c > 0, n₀ > 0 s.t. **T(n) ≥ c·g(n)** for all n ≥ n₀ | 하한 (Lower Bound) |
| T(n) = Θ(g(n)) | T(n) = O(g(n)) **AND** T(n) = Ω(g(n)) | 정확한 성장률 |

> ⚠️ **복기 주의:** g(n)이 정확히 복기되지 않음. 답이 아래와 같이 나오려면 g(n)은 n²(또는 그 이상) 으로 추정.  
> (T(n) = 2n-1은 선형이므로, g(n) = n² 기준으로 하면 O ✅, Ω ❌, Θ ❌ 가 성립)

**(a) Big-Oh: 있다 ✅**

T(n) = 2n-1 ≤ c·g(n) → 예: c = 3, n₀ = 1 → **성립**

**(b) Big-Omega: 없다 ❌**

T(n) = 2n-1 ≥ c·g(n) → g(n)이 n²이면 n이 커질수록 n² >> 2n-1 → **불가**

**(c) Big-Theta: 없다 ❌**

Big-Omega가 성립하지 않으므로 Theta도 **불가**

## 3번 — Pointer-Based List (One-step Ahead Method) 손코딩

> **배점: 약 25점**

**노드 구조:**
```cpp
struct Node {
    int data;
    Node* next;
};
```

**One-step ahead 방식의 핵심:**
- `position` 포인터는 실제 조작할 위치의 **한 칸 앞** 노드를 가리킴
- 따라서 삽입/삭제 시 이전 노드를 별도로 탐색할 필요 없음
- `head`는 더미 헤드 노드(dummy head)를 가리킴 → `head->next`부터 실제 데이터

### (1) InsertNode — position 다음에 value 삽입

```cpp
void InsertNode(Node* position, int value) {
    Node* newNode = new Node();    // 새 노드 생성
    newNode->data = value;         // 데이터 설정
    newNode->next = position->next; // 새 노드의 next = position이 가리키던 다음 노드
    position->next = newNode;      // position의 next = 새 노드
}
```

**삽입 과정 그림:**
```
Before: ... → [position] → [next] → ...
After:  ... → [position] → [newNode] → [next] → ...
```

### (2) DeleteNode — position 다음 노드 삭제

```cpp
void DeleteNode(Node* position) {
    Node* temp = position->next;     // 삭제할 노드 저장
    position->next = temp->next;     // position이 삭제 노드의 다음을 직접 가리킴
    delete temp;                     // 메모리 해제
}
```

**삭제 과정 그림:**
```
Before: ... → [position] → [temp] → [next] → ...
After:  ... → [position] → [next] → ...   (temp 해제)
```

## 4번 — 트리 함수 구현 & 구현 방식 장단점

> **배점: 약 20점**

### (1) 배열 기반 트리 함수 (1-indexed)

> **복기 메모:** 교수님 PPT 슬라이드에서 빨간 글자 부분(조건문, return 값)이 빈칸으로 출제됨.

**공식 (1-indexed 기준):**

| 함수 | 반환값 | 조건 |
|------|--------|------|
| `Parent(k)` | `k / 2` | `k ≠ 1` |
| `Left_child(k)` | `2k` | `2k ≤ n` |
| `Right_child(k)` | `2k + 1` | `2k+1 ≤ n` |
| `Left_sibling(k)` | `k - 1` | `k`가 홀수(odd) |
| `Right_sibling(k)` | `k + 1` | `k`가 짝수(even) and `k+1 ≤ n` |

```cpp
int Parent(int k)      { if (k == 1) return -1; return k / 2; }
int Left_child(int k)  { if (2*k > n) return -1; return 2*k; }
int Right_child(int k) { if (2*k+1 > n) return -1; return 2*k+1; }
int Left_sibling(int k){ if (k % 2 == 1) return k-1; return -1; } // k 홀수면 왼쪽 형제 있음
int Right_sibling(int k){ if (k % 2 == 0 && k+1 <= n) return k+1; return -1; }
```

### (2) 구현 방식 장단점

| 구현 방법 | 장점 | 단점 |
|-----------|------|------|
| **Array-based** | Cache coherence로 인해 memory access time이 작다 | 크기가 급격히 커질 때 동적 대응 어려움 (재할당·복사 비용 발생) |
| **Pointer-based** | 동적으로 크기 조절 용이 | 포인터 오버헤드로 cache coherence 낮아 access time 큼 |

> **복기 정답 그대로:**  
> - Array-based 장점: Cache Coherence 관점에서 memory access time이 작다.  
> - Array-based 단점: 사이즈가 급격하게 커졌을 때 동적 대응이 어렵다 (더 큰 배열 생성 + 복사 cost 발생).  
> - Pointer-based: Array-based의 장단점 반대.

## 5번 — Min-Heap 성질 & Heapifying CBT

### (1) 두 가지 Property 정의

**(a) Structural Property (구조적 성질):**  
Min-Heap은 **Complete Binary Tree (완전 이진 트리)** 여야 한다.  
= 마지막 레벨을 제외한 모든 레벨이 완전히 채워지고, 마지막 레벨은 왼쪽부터 채워짐.

**(b) Heap Order Property (힙 순서 성질):**  
모든 노드의 값은 자신의 자식 노드 값보다 **작거나 같아야 한다** (min-heap 기준).  
= 부모 ≤ 자식 (어떤 서브트리에서든)

### (2) Heapifying CBT (이미 주어진 CBT를 힙으로 변환)

**방법:** 마지막 내부 노드부터 루트 방향으로 각 노드에 대해 **percolate-down (sift-down)** 수행.

**시간복잡도: O(n)**

**이유:**  
- 단순히 n번 삽입하면 O(n log n)이지만,
- 아래에서 위로 heapify하면 하위 레벨 노드는 이동 거리가 짧고 개수가 많고,  
  상위 레벨 노드는 이동 거리가 길지만 개수가 적음.
- 레벨별 비용의 합이 O(n)으로 수렴 (수학적으로 Σ k / 2^k 형태).

## 6번 — Knuth Transform (Left-Child Right-Sibling)

> **복기 메모:** 변환된 이진 트리가 그림으로 주어졌음. 아래는 복기된 내용.

### 변환 규칙 (Knuth Transform)

일반 트리 → 이진 트리 변환:
- 노드의 **가장 왼쪽 자식** → **left child**
- 노드의 **다음 오른쪽 형제** → **right child**

**역변환 (이진 트리 → 일반 트리):**
- left child → 첫 번째 자식 (leftmost child)
- right child → 오른쪽 형제 (right sibling)

### PPT 예시

**원본 일반 트리:**
```
      a
    / | \
   b  c  d
  / \
 e   f
```

**변환된 이진 트리 (Left-Child Right-Sibling):**
```
      a
     /
    b ─── c ─── d
   /
  e ─── f
```
*(수직선 = left_child, 수평선 = right_sibling)*  
이진 트리 표현:
```
       a
      /
     b
    / \
   e   c
    \   \
     f   d
```

### (1) Original Tree 복원 문제

주어진 이진 트리에서 역변환 적용:
- left 포인터를 따라가면 → 첫 번째 자식
- right 포인터를 따라가면 → 형제 노드들

### (2) 핵심 성질 ⭐

> **원본 트리의 Postorder == 변환된 이진 트리의 Inorder**

**이유:** Left-Child Right-Sibling 표현에서 이진 트리를 inorder로 방문하면,  
왼쪽(자식들) → 현재 노드 → 오른쪽(형제들) 순서가 되는데,  
이는 원본 트리의 postorder(자식 먼저, 부모 나중)와 정확히 일치한다.

## 7번 — Quick Sort In-place 손코딩

> **배점: 약 20~25점**  
> **복기 메모:** 빨간 네모(빈칸) 위치 위주로 복기됨. 아래는 PPT와 유사한 표준 구현.

**피벗: 배열의 마지막 원소 사용**

```cpp
int partition(int arr[], int low, int high) {
    int pivot = arr[high];   // 피벗 = 마지막 원소
    int i = low - 1;         // ① i 초기값 (pivot보다 작은 영역의 끝 인덱스)

    for (int j = low; j < high; j++) {
        if (arr[j] <= pivot) {
            i++;
            swap(arr[i], arr[j]);  // ② arr[j] ≤ pivot인 원소를 왼쪽 영역으로 이동
        }
    }
    swap(arr[i + 1], arr[high]);   // ③ 피벗을 올바른 자리로 이동
    return i + 1;                  // ④ 피벗의 최종 인덱스 반환
}

void quickSort(int arr[], int low, int high) {
    if (low < high) {                              // ⑤ 기저 조건 (원소 1개 이하면 종료)
        int pi = partition(arr, low, high);
        quickSort(arr, low, pi - 1);               // ⑥ 피벗 왼쪽 재귀
        quickSort(arr, pi + 1, high);              // ⑦ 피벗 오른쪽 재귀
    }
}
```

**빈칸 요약:**

| 빈칸 번호 | 내용 | 답 |
|-----------|------|----|
| ① | i 초기값 | `low - 1` |
| ② swap의 역할 | pivot 이하 원소를 왼쪽으로 | — |
| ③ swap 대상 | `arr[i+1]`, `arr[high]` | `i + 1` |
| ④ return 값 | 피벗의 최종 인덱스 | `i + 1` |
| ⑤ 기저 조건 | 분할 중단 시점 | `low < high` |
| ⑥ 왼쪽 재귀 | `low` ~ `pi-1` | `pi - 1` |
| ⑦ 오른쪽 재귀 | `pi+1` ~ `high` | `pi + 1` |

**시간복잡도:**
- 평균: **O(n log n)**
- 최악: **O(n²)** — 이미 정렬된 배열에서 피벗이 항상 최솟값/최댓값일 때

## 8번 — BST 삽입 · 삭제 & Delete 함수 손코딩

> **복기 메모:** 원본 BST 그림이 주어졌으나 정확한 구조 미복기. (1)의 삽입은 17, 30. (3)의 삭제는 5, 12.

### (1) 삽입: 17과 30을 순서대로 삽입

주어진 BST에 17 삽입 후, 이어서 30 삽입한 결과를 각각 그림.

> BST 삽입 규칙: 현재 노드보다 작으면 왼쪽, 크면 오른쪽으로 재귀 탐색 후 빈 자리에 삽입.

### (2) 삽입 시간복잡도

**(a) Best Case: O(log n)**  
→ 트리가 완전한 균형(balanced) 상태일 때.  
→ 탐색 경로 = 트리 높이 = log n

**(b) Worst Case: O(n)**  
→ 정렬된 순서로 삽입하여 트리가 한쪽으로 치우친 skewed tree가 될 때.  
→ 탐색 경로 = n (연결 리스트와 동일)

### (3) 삭제: (원본 기준) 5와 12를 순서대로 삭제

> 삭제 후 각각 결과를 그림으로 표시.  
> 두 자식이 있는 경우: **중위 후계자(in-order successor)** 방식으로 처리.

**삭제 3가지 케이스:**
1. **자식 없음 (Degree 0):** 단순 제거
2. **자식 하나 (Degree 1):** 자식을 부모와 직접 연결
3. **자식 둘 (Degree 2):** 오른쪽 서브트리의 최솟값(in-order successor)으로 대체 후 그 노드 삭제

### (4) BST Delete 함수 손코딩

> **배점: 약 20~25점**  
> **복기 메모:** 인터넷에서 유사 코드 찾아 빈칸 위치 복기.

```cpp
struct Node {
    int data;
    Node* left;
    Node* right;
};

// 서브트리에서 가장 작은 노드 반환 (in-order successor 찾기용)
Node* minValueNode(Node* node) {
    Node* current = node;
    while (current->left != nullptr)
        current = current->left;     // ① 계속 왼쪽으로 내려감
    return current;
}

Node* deleteNode(Node* root, int key) {
    if (root == nullptr) return root;

    // 삭제할 노드 탐색
    if (key < root->data) {
        root->left = deleteNode(root->left, key);
    } else if (key > root->data) {
        root->right = deleteNode(root->right, key);
    } else {
        // 삭제할 노드 발견

        // Case 1 & 2: 자식이 없거나 하나인 경우
        if (root->left == nullptr) {
            Node* temp = root->right;   // ② 오른쪽 자식 저장
            delete root;
            return temp;
        } else if (root->right == nullptr) {
            Node* temp = root->left;    // ③ 왼쪽 자식 저장
            delete root;
            return temp;
        }

        // Case 3: 자식이 둘인 경우 → in-order successor 사용
        Node* temp = minValueNode(root->right);  // ④ 오른쪽 서브트리의 최솟값 (후계자)
        root->data = temp->data;                  // ⑤ 후계자 값으로 현재 노드 덮어쓰기
        root->right = deleteNode(root->right, temp->data);  // ⑥ 후계자 삭제
    }
    return root;
}
```

**빈칸 요약:**

| 빈칸 번호 | 위치 | 답 |
|-----------|------|----|
| ① | minValueNode 내 탐색 | `current->left` |
| ② | left == nullptr일 때 | `root->right` |
| ③ | right == nullptr일 때 | `root->left` |
| ④ | 후계자 찾기 | `minValueNode(root->right)` |
| ⑤ | 후계자 값 복사 | `temp->data` |
| ⑥ | 후계자 삭제 재귀 | `temp->data` |

## 시험 팁 (복기자 메모)

1. **손코딩 배점이 매우 높다** (문제당 20~30점). 코드는 교수님 PPT에서 다루지 않으므로, 아래 내용의 C++ 코드를 미리 찾아보고 손으로 써보는 연습이 필수.
   - Sorting 함수들 (Quick Sort, Merge Sort 등)
   - BST 함수 (Search, Insert, Delete)
   - Heap 함수 (Insert, DeleteMin, Heapify)
   - LinkedList 삽입·삭제 연산 함수

2. **트리 관련 PPT 슬라이드**에서 빨간색 글자(조건문, return값)가 빈칸으로 출제되므로 해당 슬라이드를 다시 확인할 것.

3. **Big-O/Omega/Theta 형식적 정의**를 수식으로 쓸 수 있어야 하고, 주어진 T(n)에 대해 c와 n₀를 구체적으로 제시해야 함.

4. **Knuth Transform 성질**: 원본 트리 Postorder = 변환된 이진 트리 Inorder — 반드시 암기.

## 핵심 공식 모음

```
[배열 기반 이진 트리 - 1-indexed]
Parent(k)       = k / 2          (k ≠ 1)
Left_child(k)   = 2k             (2k ≤ n)
Right_child(k)  = 2k + 1         (2k+1 ≤ n)
Left_sibling(k) = k - 1          (k 홀수)
Right_sibling(k)= k + 1          (k 짝수, k+1 ≤ n)

[점근 표기법]
O(g):   T(n) ≤ c·g(n)           for all n ≥ n₀
Ω(g):   T(n) ≥ c·g(n)           for all n ≥ n₀
Θ(g):   c₁·g(n) ≤ T(n) ≤ c₂·g(n) for all n ≥ n₀

[힙 배열 인덱스 - 0-indexed]
Parent(i)     = (i - 1) / 2
Left_child(i) = 2i + 1
Right_child(i)= 2i + 2

[BST 시간복잡도]
Average case: O(log n)   (균형 트리)
Worst case:   O(n)       (skewed tree — 정렬된 순서 삽입)
```
